What is React?



  • Created by Facebook, used by Instagram, Airbnb, Netflix

  • Focuses on building small, reusable components

  • Library or Framework? 🤔

📚 Library:



  • Collection of functions or modules to help solve specific tasks

  • Developers call the library code when needed (you’re in control 🛠️)

  • Typically focuses on a narrow scope of functionality

🧱 Framework:



  • Provides a skeleton or structure for your application

  • Inverts control: the framework calls your code 🤖 (“Inversion of Control”)

  • “Opinionated”: Enforces architectural patterns, limiting flexibility to ensure consistency

Which one is React?



  • Core React (react + react-dom):

    • 📚✨ A JavaScript library for building user interfaces (🧐 view layer only)

In Practice:



  • React is often used as a framework because it shapes how you build UIs with components and a declarative style

  • React Ecosystem 🔄 (React Router, Redux, Vite/Cra, etc.) transforms React into a full framework-like experience for SPAs

Summary:



  • On its own => Library 📚

  • In practice => Framework 🧱

  • End of the day: doesn’t really matter 🤷‍♀️

Why React is Popular



  • ✅ Easy to get started

  • ♻️ Component-based = Reusable code

  • ⚡ Fast updates

  • 🌍 Huge community + job market

  • 🤹 Makes JavaScript more declarative and fun

Easy to get started



  • Uses the familiar HTML, CSS, and JavaScript



💡 Think



React Components = LEGO Bricks



React + LEGOs



  • Build your UI from small pieces

  • Reuse them anywhere

  • Making development more efficient and maintainable


Fast Updates:



  • Only updates parts of the page that change

  • Instead of re-rendering entire pages

Key Features of React



  • Component-Based Architecture

  • Virtual DOM

  • Declarative UI

Component-Based Architecture



  • Apps built from small, reusable pieces

Vertical Separation of Concerns:



  • Changes the focus from separating by file type (HTML, CSS, JS) to grouping by functionality — everything for a component (markup, styles, logic) lives together in one place

  • Allows for component reusability


Virtual DOM



  • React keeps a draft (Virtual DOM)

  • Updates only the changed parts

  • Makes apps faster and smoother




Declarative UI



  • Focus on what the UI should look like, not how to update it

  • You describe what the UI should look like

  • React figures out how to make it happen

  • Simpler, readable code

What is JSX?



  • JSX = JavaScript XML

  • A syntax extension for JavaScript

  • Lets you write HTML-like code in JavaScript




      const drink = "Margarita";
      return <h1>I would like a {drink}, please!</h1>

Why JSX is awesome?



JSX Superpowers 🦸‍♀️



  • Cleaner code

  • Blend layout + logic

  • Dynamic and powerful

Cleaner code:



  • JSX looks like HTML, so it's easier to read

  • No need to switch between files

  • See HTML-like layout and your logic in same file

Dynamic:



  • JSX lets you use JavaScript expressions inside curly braces

  • Makes it easy to create dynamic content

  • Example: 


      const drink = "Margarita";
      return <h2>I'd like a {drink}, please!</h2>;

React Best Practices



  • 📁 Organize components in folders

  • 🔁 Keep components small and focused

  • 🔍 Separation of concerns: UI, logic, and state

  • ♻️ Reuse components with props

  • 💬 Add comments and meaningful names

  • 🧼 Keep JSX clean and readable
