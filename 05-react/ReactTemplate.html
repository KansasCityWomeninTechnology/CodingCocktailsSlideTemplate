State vs Props



🍸 Continuing the bartender analogy:


  • Props = What the customer asks for (external & fixed)

  • State = What's happening behind the bar (internal & changeable)

In React:



  • Props: Passed into a component from the parent

  • State: Managed within the component and can change


    • const [iceCubes, setIceCubes] = useState(10);

useState



useState to manage dynamic values like form input, toggles, and more!



🪝 useState Hook



  • Used to manage local state inside functional components

  • The array destructuring syntax gives you the state value and a function to update it


useState Hook


    import { useState } from 'react';

    function IceBucket() { 
       const [iceCubes, setIceCubes] = useState(10); 
    
       const makeDrink = () => setIceCubes(iceCubes - 2);   
       return ( 
        <div> 
          <p>Ice Cubes: {iceCubes}</p>
          <button onClick={makeDrink}>Make Drink</button> 
         </div> 

         ); 
    }

Lifting State Up



  • When multiple components need access to the same state, move the state to their closest common parent

  • 🔄 This pattern ensures consistent, shared data across components

🪝 useEffect Hook



  • Runs side effects in functional components (like fetching data)

  • ⏱️ Great for API calls, subscriptions, timers, and more

useEffect Hook




import { useEffect, useState } from 'react'; 

function App() { 
  const [drink, setDrink] = useState(null); 

  useEffect(() => { 
fetch('/api/drink') 
.then(res => res.json()) 
.then(data => setDrink(data)) 
.catch(err => console.error(err)); 
  }, []); // empty array = run once on mount 
}

& Fetching Data



What is useEffect?


  • Run code after render — fetch data, update DOM, start timers, listen for events

Example:



Make a Cocktail When Selected


useEffect(() => { 
  console.log(`Making a ${selectedDrink}... 🍹`);
}, [selectedDrink]);

Run Once on Mount


useEffect(() => {
  console.log('Setting up the cocktail station...');
}, []);

Cleanup Example




useEffect(() => { 
  const interval = setInterval(() => { 
    console.log('Shaking the cocktail shaker... 🍸');
  }, 1000); 

  return () => { 
   clearInterval(interval);
   console.log('Bar is closed. No more shaking.'); 
  }; 
}, []);

Try/Catch Blocks


What is try/catch?


  • Try to run code

  • Catch errors to prevent crashes

Cocktail Example



function MixCocktail({  ingredients }) { 
  try { 
   if (!ingredients.includes('lime')) 
 {throw new Error('We’re out of lime! 🍋');
    } 
    return <p>Here’s your perfectly mixed Mojito! 🍃</p>; 

  } catch (error) { 
    return <p>Oops! {error.message}</p>; 
  } 
}
