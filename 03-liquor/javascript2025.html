Foundations

of JavaScript


  • Can change HTML content and CSS styles on a webpage

  • Reacts to user interactions

  • Can send network requests

  • Can also be used for server-side programming with Node.js

History

of JavaScript


  • Developed in 1995 at Netscape

  • Created by Brendan Eich in 10 days

  • ECMAScript is the official language

  • JavaScript is one implementation of the standard

History

of JavaScript Names


  • Mocha:
    Original name of JavaScript

  • LiveScript:
    Temporary name before final release

  • JavaScript:
    Final name

JavaScript

vs Java


JavaScript and Java are two different programming languages that are often confused due to their similar names.

Key Differences


  • JavaScript: Developed at Netscape
  • Java: Developed at Sun Microsystems


  • JavaScript: Primarily Client-Side [Node.js allows for server-side programming]
  • Java: Server-Side [Java applets allow for client-side programming]


  • JavaScript: Loosely Typed: (Variables can hold any type of data and can be reassigned to different types)
  • Java: Strictly Typed: (Variables must be declared with a specific type and cannot be reassigned to a different type)

ES6

(ECMAScript 2015)


  • 2015 - new version of JavaScript was released called ES6 (or ES2015)

  • Major version that introduces many new features into JavaScript

  • Since ES6, a new version of JavaScript has been released every June

  • As of June 2026, we will be on ES17 (ES2026)

How is

JavaScript Used?


Linking

JavaScript Files



  • Usually in <head>, but can also be added before closing </body>

  • Use the <script> tag

  • Use the src attribute to link to the file

  • Use the type attribute to specify the language

How to

add JavaScript



  • You can write JavaScript directly inside an HTML file


    • <script> alert("Hello, JavaScript!"); </script>


  • Most of the time, it is better to link to a separate JavaScript file


    • <script src="assets/js/my-javascript-file.js"></script>

DOM

(Document Object Model)


  • A webpage is a document

  • Browser parses HTML to create the DOM

  • DOM: Object-oriented representation of the webpage

  • JS can manipulate DOM

How Does

JavaScript Manipulate the DOM?



  • Changing colors

  • Add/change/remove text

  • Animate HTML elements

Variables


Variables



  • Placeholder to store values/data

  • const and let : good practice is to always set a variable using one of these keywords

  • var : avoid using this keyword since ES6

Const & Let


  • const
    variable that cannot be reassigned

  • let
    variable that can be reassigned

    • let myName = "John";
    myName = "Jane"; // "Jane"


    const myName = "John";
    myName = "Jane"; // TypeError: Assignment to constant variable.

Variable

Naming


  • JavaScript is case-sensitive:
    use same case you used when creating the variable to reference said variable

  • myVariable is not the same as MyVariable

Variable

Naming Rules


  • Must start with a letter [upper or lowercase], _, or $

  • Subsequent characters may contain letters, numbers, _, or $

  • Best practice:
    camelCase

Camel Case



  • First word is lowercase

  • Subsequent words have have their first letter capitalized, followed by all lowercase

  • const myVeryFirstVariable

Example


  • Set a variable

    • const myVariable = 5;
  • Reference that variable by name

    • myVariable;


Data

Types


Common

Data Types


  • Strings

  • Booleans

  • Numbers

  • Undefined

  • Null

Strings


  • Sequence of characters

  • Enclosed in single or double quotes

  • Able to concatenate

Example


const double = "Hello World!";


const single = 'Hello World!';


typeof double; // string

Concatenation


  • Joining strings together

  • Uses + operator


const hello = "Hello";


const world = "World!";


const helloWorld = hello + " " + world;


// "Hello World!"

Booleans


  • True or False

  • Used in conditional statements


const isTrue = true;
const isFalse = false;
typeof isTrue; // boolean

Numbers


  • Can be whole numbers or decimals

  • Can use arithmetic operators

  • Can use comparison operators

Example


const num = 5;


const decimal = 5.5;


typeof num; // number

Fun fact 🧐



  • Wrapping a number in quotes turns it into text
    like writing the word "five" instead of the number 5

const num = "5";
typeof num; // string

Undefined


  • undefined means a variable exists, but it has not been given a value yet

  • This often happens when you declare a variable and assign a value later
let x;
typeof x; // undefined

Null



  • Use null when you want to intentionally say, "this variable has no value right now"

  • null is its own primitive value, even though this can look confusing:

    • let x = null;
    typeof x; // object
  • typeof null returns "object" because of a legacy bug in JavaScript

Operators


  • Math (Arithmetic)

  • Comparison

  • Logical

Operators

Math (Arithmetic)


  • Addition/Subtraction

  • Multiplication/Division

  • Modulus (Similar to remainder)

  • Incrementation

  • Addition (+)

    • const myAge = 21 + 10; // 31
  • Subtraction (-)

    • const myDogsAge = 31 - 20; // 11

  • Increment (++)

    • let myAge = 21;
    myAge++; // 22
  • Decrement (--)

    • let cookies = 5;
    cookies--; // 4

Operators

Comparison


  • Loose equal to (==)

    • const myAge = 21;
    myAge == 21; // true
  • Strict equal to (===): checks value and type

    • const cookies = 0;
    const drinks = '0';
    cookies === drinks; // false

Loose Equality

vs Strict Equality



  • == operator allows type coercion

  • === operator compares both value and type

Functions



What is

a Function?



  • Reusable block of code

  • Helps reduce duplication (DRY)

  • Can accept inputs (parameters) and return outputs


Reusable

Code



  • Instead of writing the same logic over and over...

  • Place logic in a function and use that function over and over

DRY

programming



  • Don't

  • Repeat

  • Yourself

Function

Declaration



function greet() {
    return `Welcome, Coders!`;
}

// Call the function
greet();

More About

Functions


Function

Parameters


  • Let you create flexible, reusable code

  • Functions can accept 0 to many parameters, separated by commas

  • Example:
    • const greet = function(name) {
        return `Hello, ${name}!`;
    };

Function

Expression



// assign a function to a variable
const orderDrink = function(drinkName) {
  return 'I would like a ' + drinkName;
};

// call the function
orderDrink('gin and tonic');
// 'I would like a gin and tonic'

Arrow

Functions


  • ES6 introduced arrow functions as a shorter way to write functions

  • Arrow functions can take parameters and return values, just like regular functions

  • One key difference: arrow functions do not create their own this value like regular functions do

  • They are often assigned to a variable, but that is not what makes them an arrow function

Conditionals



  • JavaScript executes from top to bottom (synchronously)

  • Conditionals allow us to control the flow of our code

  • Sometimes you only want to execute code if something is true or false

  • JavaScript has different types of conditionals with the most common being if/else

if/else



  • Statements used to execute code if a condition is true or false

  • Can be nested together to create more complex logic

if (condition) {
  // code to execute if condition is true
} else {
  // code to execute if condition is false
}

Example



const age = 21;

if (age >= 21) {
  console.log('You can drink!');
} else {
  console.log('You can't drink!');
}
// You can drink!

Else if



  • else if lets you check another condition


    • if (condition) {
      // code to execute if condition is true
    } else if (anotherCondition) {
      // code to execute if the next condition is true
    }

Else



  • else is the fallback when no earlier condition matches


    • if (condition) {
      // code to execute if condition is true
    } else if (condition) {
      // code to execute if first condition false, but second true
    } else {
      // code to execute if condition is false
    }

Ternary

Operator



  • A shorthand way to choose between two values based on a condition

  • condition ? value if true : value if false ;

    • const myAge = 18;
    const canDrink = myAge >= 21 ? 'Drink!' : 'Not yet!';
    // 'Not yet!'

console.log()



  • Prints output to the console

  • Useful for debugging

  • Can be used to print out the value of variables
const baskets = 5;
const balls = 13;
const numberOfBallsPerBasket = balls / baskets;

console.log('Balls per basket:', numberOfBallsPerBasket);
// Balls per basket: 2.6

Arrays

& Loops


Arrays



  • An array lets you store multiple values in a single variable

  • Array items are written inside square brackets and separated by commas

  • Each item has a position called an index, starting at 0

Array

Examples



const cocktails = ['Code on the Beach', 'jQuery Sour'];


const drinksIveConsumed = [0, 1, 2, 3, 4, 5, 6, 7, 'none of the above'];

Create

an Array



  • You can create an array with new Array()

  • Most often, you will create an array with square brackets: []

    • const cocktails = new Array(); // works
    const drinksIveConsumed = []; // most common

Zero-based

Indexing



  • You access items in an array by their index, or position

  • Because arrays are zero-based, the first item is at index 0

  • That means the second item is at index 1

  • The last item is always at array.length - 1

Accessing

Items



const cocktails = ['Code on the Beach', 'jQuery Sour', 'Silicon Valley', 'JS on Rocks'];

console.log(cocktails[0]);
// Code on the Beach

console.log(cocktails[1]);
// jQuery Sour

console.log(cocktails[cocktails.length - 1]);
// JS on Rocks

What Can You Do

with an Array?



  • Access items by index

  • Loop through all the items

  • Add or remove items

Loops



  • A loop lets you repeat a block of code

  • JavaScript has several kinds of loops, but we will focus on the for loop

For Loop

Structure



  • initialization sets the starting point

  • condition is checked before each loop

  • increment updates the value each time through the loop


  • When the condition becomes false, the loop stops


    • for (initialization; condition; increment) {
      // code to execute while condition is true
    }


forEach()



  • forEach() is an array method for looping through array items

  • It runs a function once for each item in the array

  • It is a convenient option when you want to work with every item in an array

for Loop



// using a for loop
const cocktails = ['Code on the Beach', 'jQuery Sour'];

for (let i = 0; i < cocktails.length; i++) {
  console.log(cocktails[i], i);
}

// Code on the Beach 0
// jQuery Sour 1

forEach()



// using forEach()
const cocktails = ['Code on the Beach', 'jQuery Sour'];

cocktails.forEach(function(cocktail, index) {
  console.log(cocktail, index);
});

// Code on the Beach 0
// jQuery Sour 1

Accessibility

Accessibility

with JavaScript



  • Why accessibility matters

    • JavaScript often controls menus, modals, tabs, and other interactive UI

    • That means JavaScript can improve accessibility or accidentally break it


  • When adding interactivity, make sure the experience still works for keyboard and assistive technology users

Focus &

Keyboard Access



  • Use real interactive elements like <button> whenever possible

  • If JavaScript opens something like a menu or modal, move focus to the right place

  • Users should be able to use interactive features without a mouse

Focus

Example



openButton.addEventListener('click', function() {
  dialog.hidden = false;
  closeButton.focus();
});

Dynamic

Updates



  • If JavaScript changes content on the page, users need a clear way to notice that change

  • This includes clear visual feedback and, when needed, helping screen readers announce updates

  • Good accessible JavaScript preserves meaning, focus, and usability as the page changes

Intro to

TypeScript




Intro to

TypeScript



  • TypeScript is JavaScript with extra type checking

  • You can add types to variables, function parameters, and return values

  • That helps catch certain mistakes before the code runs


  • You write TypeScript, but the browser still runs regular JavaScript

  • TypeScript checks your code, then compiles it to JavaScript

  • Types can be inferred automatically or written explicitly

Type

Inference



let drinkName = 'Mojito'; // inferred as string
drinkName = 42; // Error

let isMocktail = true; // inferred as boolean

Explicit

Types



let drinkName: string = 'Mojito';
let guestCount: number = 28;
let isMocktail: boolean = true;

Why Use It?



  • It helps catch type mistakes earlier

  • It makes code easier to read because expected values are clearer

  • It becomes especially helpful as projects grow

Think of it as JavaScript
with type checking
