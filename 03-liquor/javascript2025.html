Foundations

of JavaScript


  • Able to manipulate HTML content & CSS styling on a webpage

  • Reacts to user interactions

  • Can send network requests

  • Can be used for server-side programming (Node.js)

    • History

    of JavaScript


    • Developed in 1995 at Netscape

    • Created by Brendan Eich in 10 days

    • ECMAScript is the official language

    • JavaScript is one implementation of the standard

    History

    of JavaScript Names


    • Mocha:
      Original name of JavaScript

    • LiveScript:
      Temporary name before final release

    • JavaScript:
      Final name

    JavaScript

    vs Java


    JavaScript and Java are two different programming languages that are often confused due to their similar names.

    Key Differences


    • JavaScript: Developed at Netscape
    • Java: Developed at Sun Microsystems


    • JavaScript: Primarily Client-Side [Node.js allows for server-side programming]
    • Java: Server-Side [Java applets allow for client-side programming]


    • JavaScript: Loosely Typed: (Variables can hold any type of data and can be reassigned to different types)
    • Java: Strictly Typed: (Variables must be declared with a specific type and cannot be reassigned to a different type)

    ES6

    (ECMAScript 2015)


    • 2015 - new version of JavaScript was released called ES6 (or ES2015)

    • Major version that introduces many new features into JavaScript

    • Since ES6, a new version of JavaScript has been released every June

    • As of June 2026, we will be on ES17 (ES2026)

    JavaScript Used?

    JavaScript Files



    • Usually in <head>, but can also be added before closing </body>

    • Use the <script> tag

    • Use the src attribute to link to the file

    • Use the type attribute to specify the language

    Adding JavaScript



    • Add JavaScript directly to files

      • 
<script>

    BEST PRACTICE

    Link to JS File


    <script src="assets/js/my-javascript-file.js"></script>

    (Document Object Model)


    • A webpage is a document

    • Browser parses HTML to create the DOM

    • DOM: Object-oriented representation of the webpage

    • JS can manipulate DOM

    How Does JavaScript

    Manipulate the DOM?


    • Changing colors

    • Add/change/remove text

    Variables


    Variables



    • Placeholder to store values/data

    • const and let : good practice is to always set a variable using one of these keywords

    • var : avoid using this keyword since ES6

    Const & Let


    • const : variable that cannot be reassigned

    • let : variable that can be reassigned


    let myName = "John";
    myName = "Jane"; // "Jane"


    const myName = "John";
    myName = "Jane"; // TypeError: Assignment to constant variable.

    Variable

    Naming


    • JavaScript is case-sensitive: use same case you used when creating the variable to reference said variable

    • myVariable is not the same as MyVariable

    Variable

    Naming Rules


    • Must start with a letter [upper or lowercase], _, or $

    • Subsequent characters may contain letters, numbers, _, or $

    • Best practice: camelCase

    Camel Case



    • First word is lowercase

    • Subsequent words have have their first letter capitalized, followed by all lowercase

    • const myVeryFirstVariable

    Example


    • Set a variable

      • const myVariable = 5;
    • Reference that variable by name

      • myVariable;


    Primitive

    Values


    Common

    Data Types


    • Strings

    • Booleans

    • Numbers

    • Undefined

    • Null

    Strings


    • Sequence of characters

    • Enclosed in single or double quotes

    • Able to concatenate

    Example


    const double = "Hello World!";


    const single = 'Hello World!';


    typeof double; // string

    Concatenation


    • Joining strings together

    • Uses + operator


    const hello = "Hello";


    const world = "World!";


    const helloWorld = hello + " " + world;


    // "Hello World!"

    Booleans


    • True or False

    • Used in conditional statements


    const isTrue = true;
    const isFalse = false;
    typeof isTrue; // boolean

    Numbers


    • Also commonly called integers

    • Can include decimals

    • Can use arithmetic operators

    • Can use comparison operators

    Example


    const num = 5;


    const decimal = 5.5;


    typeof num; // number

    Fun fact 🧐



    • Wrapping a number in quotes turns it into text
      like writing the word "five" instead of the number 5

    const num = "5";
    typeof num; // string

    Undefined


    • A variable that has been declared, but has no value

    • Can be assigned a value later
    let x;
    typeof x; // undefined

    Null


    • Explicitly set a variable to null, as a representation that the variable has no value
    let x = null;
    typeof x; // object
    • Due to a legacy bug in the language.

    Operators


    • Math (Arithmetic)

    • Comparison

    • Logical

    Math (Arithmetic)

    Operators


    • Addition/Subtraction

    • Multiplication/Division

    • Modulus (Similar to remainder)

    • Incrementation

    • Addition (+)

      • const myAge = 21 + 10; // 31
    • Subtraction (-)

      • const myDogsAge = 31 - 20; // 11

    • Increment (++)

      • let myAge = 21;
      myAge++; // 22
    • Decrement (--)

      • let cookies = 5;
      cookies--; // 4

    Comparison

    Operators


    • Equal to (==)

      • const myAge = 21;
      myAge == 21; // true
    • Strict equal to (===): type specific

      • const cookies = 0;
      const drinks = '0';
      cookies === drinks; // false

    Loose Equality

    vs Strict Equality



    • == operator allows type coercion

    • === operator compares both value and type

    Functions



    What is a Function?



    • Reusable block of code

    • Helps reduce duplication (DRY)

    • Can accept inputs (parameters) and return outputs


    Reusable Code



    • Instead of writing the same logic over and over...

    • Place logic in a function and use that function over and over

    DRY

    programming



    • Don't

    • Repeat

    • Yourself

    Function Declaration



    function greet() {
        return `Welcome, Coders!`;
    }

    // Call the function
    greet();

    Advanced Functions


    Function

    Parameters


    • Let you create flexible, reusable code

    • Functions can pass 0 to many parameters (separated by commas)

    • Example:
      • const greet = function(name) {
          return `Hello, ${name}!`;
      };

    Set to a Variable Value



    //with 1 parameter
    const orderDrink = function(drinkName) {
      return 'I would like a ' + drinkName;
    };

    orderDrink('gin and tonic');
    //'I would like a gin and tonic';

    ES6 Arrow Functions



    • ES6 introduced arrow functions

    • "Regular" functions and Arrow functions are mostly the same

    • Difference is that regular functions have their own context, which arrow functions do not

    • Essentially, we're saving the function as a variable

    Conditionals



    • JavaScript executes from top to bottom (synchronously)

    • Conditionals allow us to control the flow of our code

    • Sometimes you only want to execute code if something is true or false

    • JavaScript has different types of conditionals with the most common being if/else

    if/else



    • Used to execute code if a condition is true or false

    • if/else statements can be nested together to create more complex logic

    if (condition) {
      // code to execute if condition is true
    } else {
      // code to execute if condition is false
    }

    Example



    const age = 21;

    if (age >= 21) {
      console.log('You can drink!');
    } else {
      console.log('You can't drink!');
    }
    // You can drink!

    Else & Else If


    • else (optional)
      • if (condition) {
        // code to execute if condition is true
      }
    • Else if
      • if (condition) {
        // code to execute if condition is true
      } else if (condition) {
        // code to execute if first condition false, but second true
      } else {
        // code to execute if condition is false
      }

    Ternary Operators



    • Condition ? expr1 : expr2;

    • {condition} ? {what happens if true} : {what happens if false} ;

      • const myAge = 18;
      const canDrink = myAge >= 21 ? 'Drink!' : 'Not yet!'; // Drink!

    console.log()



    • Prints output to the console

    • Useful for debugging

    • Can be used to print out the value of variables

    Arrays and Loops



    Arrays



    • A list of data items

    • Items are separated by commas

    • Items can be accessed by index

    Array Examples



    const cocktails = ['Code on the Beach', 'jQuery Sour'];


    const drinksIveConsumed = [0, 1, 2, 3, 4, 5, 6, 7, 'none of the above'];

    Create an Array



    • Use new Array()

    • Initialize with an empty string []

      • const cocktails = new Array();
      const drinksIveConsumed = [];

    Zero-based Index



    • Grab items in the array via their index

    • First item in an array is at index 0

    • Second item in an array is at index 1

    • Last item is one less than the length of the array (number of items in the array)

    Array Example



    const cocktails = ['Code on the Beach', 'jQuery Sour', 'Silicon Valley', 'JS on Rocks'];

    console.log(cocktails[0]);
    // Code on the Beach

    console.log(cocktails[1]);
    // jQuery Sour

    console.log(cocktails[cocktails.length - 1]);
    // JS on Rocks

    Things you can do to an Array



    • Loop through an array

    • Add / Remove items from the array

    • And more...

    Loops



    • A way to repeat a block of code, as long as specified condition returns true

    • JS supports multiple types of loops, but we'll cover the 'for' loop today

    For Loops Structure



    • Increment increases the value set in initialization

    • Eventually condition is false so that...

    • The loop stop executing

    • (Don't get caught in an infinite loop!)

    for (initialization; condition; increment) {
      // code to execute while condition is true
    }


    For Each



    • A loop that iterates through an array

    • An array method that executes a given function once for each array element

    Loops Example

    const cocktails = ['Code on the Beach', 'jQuery Sour'];

    for (let i = 0; i < cocktails.length; i++) {
      console.log(cocktails[i], i);
    }

    // Code on the Beach 0
    // jQuery Sour 1
    cocktails.forEach(function(cocktail, index) {
      console.log(cocktail, index);
    });

    // Code on the Beach 0
    // jQuery Sour 1

    TypeScript



    TypeScript



    • TypeScript is a superset of JavaScript

    • It adds optional static typing

    • Helps catch bugs before code runs

    • Compiles down to regular JavaScript

    • Types are either inferred or explicitly declared (with things like interfaces and types)

    TypeScript Example: Implicit



    let userName = 'Heidi'; // string
    userName = 28; // ERROR

    let isMentor: boolean = true;

    TypeScript Example: Explicit



    let userName: string = 'Heidi';
    let age: number = 28;


    let isMentor: boolean = true;

    💡 Think of TypeScript as JavaScript with superpowers!



    💡 Or a pain in the butt...





    But it can be worth it for larger projects!

    Accessibility



    Accessibility

    Things to Consider



    • Many JS features rely on mouse interactions

    • Makes it difficult for keyboard users to navigate

    • Key Idea: Not everyone interacts with a website the same way

    Accessibility

    Things to Consider



    • Functions should support clear feedback

    • Does the user know something happened?

    • Don't rely only on visual cues

    • Think: Could a screen reader detect this?

    Think Beyond the Code

    Things to Consider



    • As you learn how to code, remember

    • You're not just writing code

    • You're shaping experiences
