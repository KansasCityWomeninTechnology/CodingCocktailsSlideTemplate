Conditionals

Flow Control

  • JavaScript runs from top to bottom
  • You want to submit a drink order request (form) ONLY IF the user has entered their name

If/Else

if (expressionInParens) {
  // do something if true
}
else {
  // otherwise, do something else
}

Programming Nugget: `else` is optional.

It is possible to use an `if () {}` statement by itself.

If/Else Example

            var usersName = 'Heidi';
if (usersName.length > 0) {
  submitDrinkOrder();
}
else {
  alert('Please enter your name 
  before submitting your drink order.');
}

Falsy Values

The following are always FALSE

  • false
  • undefined
  • null
  • 0
  • NaN
  • an empty String: ""

any other value is always TRUE

Loops

Loops

  • Repeat some code
  • As long as some condition is true
  • Different kinds of loops:
    • while
    • do...while
    • for...in/of/statement
    • Today we will learn about the 'for' loop

for Loop

            for (initialization; condition; increment) {
// code to execute while condition is true
}
  • increment increases the value set in initialization
  • so that eventually the condition is false & the for loop stop executing
  • with a few exceptional cases, you DO want your loop to STOP eventually

for Loop

var numberOfCustomers = 5;
for (var i = 0; i < numberOfCustomers; i++) {
  notifyDrinkIsReady(i);
}

one of these will never terminate & is an 'infinite loop' 

var numberOfCustomers = 5;
for (var i = 0; i < numberOfCustomers;) {
  notifyDrinkIsReady(i);
}

Arrays: how to make one

When you want a collection of data items or even functions

var cocktails = ['Code on the Beach', 'jQuery Sour'];
var drinksIveConsumed = [0, 1, 2, 3, 4, 5, 6, 
  7, 'none of the above'];
var anArrayOfBooleans = [true, true, true, false];

Create a new array with 'new Array()' or just initialize it to '[]' which is an empty array.


          var myNewArray = new Array();
          var myEmptyArray = [];

Arrays: stuff you can do

  • You can grab items from the array via their index
  • Arrays have a zero-based index
  • To get the first item you use 0, to get the last item you subtract 1 from the number of items in the array
var first = cocktails[0] 
// this will return 'Code on the Beach'
var last = cocktails[cocktails.length - 1]; 
// returns 'jQuery Sour'

Arrays: stuff you can do

You can loop over an array:

var cocktails = ['Code on the Beach', 'jQuery Sour'];
// array 'for loop'
  cocktails.forEach(function(cocktail,index)){
  console.log(cocktail,index);
}
// 'Code on the Beach' 0
// 'jQuery Sour' 1

Arrays: stuff you can do

Add an item to the end of an array:

cocktails.push('Mojito');
// cocktails now looks like this: 
// ['Code on the Beach', 'jQuery Sour', 'Mojito']

Remove an item from the end of an array:

cocktails.pop();
// cocktails now looks like this:
// ['Code on the Beach', 'jQuery Sour']

Last-In-First-Out aka LIFO

Arrays: stuff you can do

Add to the beginning of an array

cocktails.unshift('JavaScriptini');
// ['Javascriptini', 'Code on the Beach', 'jQuery Sour']

Remove from the beginning of an array

cocktails.shift();
// ['Code on the Beach', 'jQuery Sour']

Objects

  • Sometimes you have complex data
  • Objects allow you to structure & organize this data

Objects: how to make one

  • Create an object with 'new Object()' or initialize an empty one with '{}'
  • Use JSON (JavaScript Object Notation) to initialize
    • 'key':'value' pairs -- "key" may also be referred to as 'name' or 'property'
    • Most API’s are going to be in this format
    • Don’t know what API’s are? resource here: http://readwrite.com/2013/09/19/api-defined/)

Objects: how to make one

var myObject = {
  "keyWrappedInQuotes": *,
  "valueCanBeAString": "string",
  "valueCanBeANumber’: 3,
  "valueCanBeABoolean": true,
  "valueCanBeAnArray": ["Red:, "White", "Sangria"],
  "valueCanBeAnObject": {
    "keyInsideChildObject": "is this inception?"
  }
  "valueCanBeAFunction": function() {
    shutTheFrontDoor();
  }
}

Objects: how to get data

  • You can get data from the Object by the key
  • Dot Notation
    • 
            myObject.valueCanBeAnArray
            // ['Red', 'White', 'Sangria']

            myObject.valueCanBeAnObject.keyInsideChildObject
            // 'is this inception?'
  • Property name has be alphanumeric, can contain underscores & can't start with a number. No spaces.

Objects: how to get data

  • You can get data from the Object by the key
  • Bracket Notation
    • 
myObject['valueCanBeAnArray']
// ['Red', 'White', 'Sangria']
myObject['valueCanBeAnObject']['keyInsideChildObject']
  • Can be used for property names that don't work with dot notation.
    • 
var objectWithSpaces = {"Uptown Funk": "You Up"};
objectWithSpaces['Uptown Funk'];
// "You Up"

Objects: how to get data

  • When you are doing something dynamic like
    • using a for loop to access elements
    • passing a property name as a parameter to a function
  • you have to use Bracket Notation
    • 
var objectKey = 'Downtown';
var songLyrics = {"Downtown": 'Banana seat, a canopy on two wheels'};
songLyrics[objectKey];
// 'Banana seat, a canopy on two wheels'

Objects: how to set a value

var myFavoriteThings = {};
myFavoriteThings.animal = 'giraffe';
console.log(myFavoriteThings);
// {"animal": 'giraffe'}
var cocktailsForMenu = [
  {
    "name": "Whiskey Sour",
    "price": "$13"
  },
  {
    "name": "Hug from Ryan Gosling",
    "price": "$15"
  }
]
cocktailsForMenu[0]['name']
// 'Whiskey Sour'
cocktailsForMenu[1].name
// 'Hug from Ryan Gosling'

Fetching with APIs

Static vs Dynamic Applications

  • Static Application
    • written in HTML (maybe CSS)
    • will only change if you edit the HTML (or CSS) 'by hand'
  • Dynamic Application
    • written in some programming language (JavaScript, PHP, Java ...)
    • content is pulled from a database/internet often triggered by user interaction
    • client-side JavaScript uses APIs to fetch data from the internet

API

  • Application Programming Interface
  • it's the middle-man that allows 2 applications to talk to each other.
  • One application makes a request.
  • API takes request to other application & gets response.
  • API brings response back to original application.

API

Weather REST API

API

  • In a web application...
  • API takes request from front end app: 'Get weather in Paris in Celcius'
  • API has its own server which will give a response: 'Forecast for Paris: 16 low, 23 high'
  • API gives this response to the frond end app
  • The front end app displays the response with fancy HTML/CSS

API: famous examples

  • Giphy [Find gifs or stickers]
  • NASA Api [for amazing imagery]
  • Google Maps [API to display/interact with Google Maps]
  • Twitter [to read or send tweets]
  • Pinterest [to view pins or add pins]
  • Some Public APIs

API: Interaction with JavaScript

  • The original way JavaScript interacted with API requests was through `XHR` [XMLHttpRequest].
  • Then jQuery came along & provided an easier interaction with `AJAX` [Asynchronous JavaScript & XML].
  • ECMAScript 2015 [was ES6] introduced `fetch` a native JS function that is similar to jQuery's AJAX.

API: fetch

  • That means you can use `fetch` without having to include a BULKY library.
    • fetch(url) // URL of API
  .then(function() {
    // Code for handling data response from API
  }
  .catch(function() {
    // Code to run if API returns an error
  });

API: fetch & format with JSON

  • There are different formatting methods, but most of the time you are going to want the format in JSON
  • APIs may have various options for you to select from.
  • For example:`http://ron-swanson-quotes.herokuapp.com/v2/quotes` -> returns 1 quote
  • `http://ron-swanson-quotes.herokuapp.com/v2/quotes/5` -> returns 5 quotes

API: fetch & format with JSON

var url = 'http://ron-swanson-quotes.herokuapp.com/v2/quotes';
fetch(url)
  .then(function(response){
    return response.json();
  })
.then(function(response){
    console.log(response);
    // ["Great job, everyone. The reception will be held in 
    // each of our individual houses, alone."]
  })

  .catch(function(error){
    console.log('error');
  });

Templates

Templates

  • Raw API responses are not very exciting to look at
  • We want to make our HTML/CSS markup use dynamic content from the API responses
  • A template is a reusable chunk of code used for markup
  • It contains placeholders [of variables] for where the dynamic content is populated.

Templates: different ways of using them

  • Library
    • A library is a set of tools that help you write JavaScript faster, but provide more flexibility in terms of architecture
    • Handlebars.js
    • Mustache.js
    • doT.js

Templates: different ways of using them

  • Framework
    • It provides a full suite of tools & helpers, with a specific structure to how you write your code
    • Frameworks have their own way of templating, that is specific to the framework (covered in Single Page App Session)
    • React
    • Angular

Templates: different ways of using them

  • Native JavaScript
    • With ECMAScript 2015, we now have template literals.
    • Provides an alternative to concatenating strings, which allows for us to explore templating with native JS.
    • This is what we'll be using tonight.

Templates: template literals

  • Can be multi-line strings wrapped with backticks OR can use expressions with the syntax `${}`.
  • Expressions can be variables, data from arrays/objects or even contain logic.

Templates: template literals

var caliberOfDay = 'sunshiney';
var myMultiLineString = `
  Hello!  
  Hope you have a ${caliberOfDay} day!`;

VS

var myMultiLineString = 'Hello!' + \n
'Hope you have a ' + caliberOfDay + ' day!';

Templates: template literals

Expression with some logic in a template

`I should drink ${8 - 5} 
          more glasses of water today.`
// I should drink 3 more 
// glasses of water today.
