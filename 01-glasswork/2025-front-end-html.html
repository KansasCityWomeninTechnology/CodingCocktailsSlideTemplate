What is

Web Development?



  • How websites and web apps are built and maintained 🧱

  • Involves writing code that browsers understand and turns into webpages you can see and use 🌐

What is a website?



  • Collection of files stored on a server that your browser requests when you visit a URL 🗃️

  • Files include code and media that browsers interpret and display as web pages 🖥️

  • Combine content, design, and interactivity to create rich experiences ⚙️

Think 🤔:



  • Website = digital book 📖:
    Server holds the pages, browser flips through and shows them to the user


Static vs. Dynamic



  • Static =
    basic pages that don’t change 📄

  • Dynamic =
    pages that update and respond to user input 💬

From URL to Website



  • Type a web address into your browser 👩‍💻
    codingandcocktails.kcwomenintech.org

  • Browser sends request to a server 📡

  • Server replies with the website’s files 🗂️

  • Browser reads the code files and displays the site content 💻

Think 🤔:




Browser-server handshake 🤝

Code Files



  • HTML:
    Defines the structure and organizes the content (navigation, header, footer, lists) 💀

  • CSS:
    Styles the content (colors, fonts, layout) 👩

  • JavaScript:
    Handles interactions (form inputs, button clicks) and fetches up to date dynamic content 🤹

Additional Files



  • Images 🎞️

  • Videos 📼

  • Music 📻

  • Fonts ✍️

  • Libraries 📚

  • Frameworks 🏗️

Code Files =

Housing Blueprints



How do they

work together?



  • HTML = structure
    (walls, floors, and foundation)

  • CSS = style
    (paint, decorations, and design)

  • JavaScript = functionality
    (electricity, plumbing, smart tech, anything interactive)


Summary 🧐



  • HTML :
    content you see 👀

  • CSS :
    styles and layout for the site 🎨

  • JavaScript :
    defines behavior of the website 🕹️

HTML Basics



  • Most essential component of webpages 🔑

  • Language with specific rules that tells the browser how to display the content 📘

  • Defines the meaning and structure of the content (not appearance) 📖


HTML



    Hyper


    Text


    Markup


    Language

🕺 HTML Breakdown



  • HyperText:
    Links that connect web pages together 🔗

  • Markup:
    Tags that describe the structure of content 🏗️

  • Language:
    Set of machine readable rules that browsers understand to display webpages 🌐

HTML Tags



HTML Tags



  • 🗣️ Tell browser how to display content

  • MOST cases require a pair:

    • Opening and closing tags

Open/Close Tags



  • Opening tag starts an element

    📗OpeningTag📗

  • Closing tag ends an element

    📘/ClosingTag📘

  • Content goes between opening and closing tags

    📗Content /📘

  • Opening tag has tag name in angle brackets (< >)

  • Closing tag has forward slash ( / ) before tag name (< />)

Self-closing Tags



  • 🙅‍♀️ Unable to add more HTML inside of tags

  • Most common: Image <img /> 🎞️

Other common Self-closing Tags:



  • <br> Line break ↩️

  • <hr> Horizontal rule (divider line) ➖

  • <input> Form input field 📝

  • <embed> Embeds external content (video, PDF) 💽

  • <source> Specifies media resources
    (<audio> & <video>) 🎧

❗ Required HTML Tags



Every HTML page requires 4 tags in this order:




<!DOCTYPE html>
<html>
  <head>...</head>
  <body>...</body>
</html>

!DOCTYPE html Tag



<!DOCTYPE html>



  • 🔝 Very top of the HTML document

  • Tells browser what version of HTML is being used

  • Current version: HTML5

Fun fact 🧐



  • HTML = “living standard”
    meaning it’s continuously updated
    (Technically no “HTML6”)


HTML Tag



<html>



  • 🏁 Starts beginning of HTML document

  • Wraps <head> & <body> elements

Head Tag



<head>



  • ☝️ First element inside an <html> tag

  • Gives the browser important information about how to interpret the HTML on the page

  • Content invisible to the user 🙈


Other valid tags inside of <head>:



  • <title> Text that displays on browser tab 🗂️

  • <meta charset="utf-8"> Character encoding:   HTML5 contains almost all characters & symbols in the world 🌏

More valid tags inside of <head>:



  • <meta> Metadata 🧬
    (search engine optimization, mobile responsiveness)

  • <link> Linked files 🔗
    (CSS Stylesheets, JavaScript Files)

Linked File Tag

Examples:



  • Add CSS linked files:
    <link rel="stylesheet" href="/css/style.css"></link>


  • Add JavaScript linked files:
    <script src="/javascript-file.js"></script>

Body Tag



<body>



  • ✌️ Second element inside of a <html> tag

  • Content IS visible to the user 👩‍💻

  • What you see when visiting a website lives in the <body>tag


Summary 🧐



  • HTML is the backbone of web pages 🩻

  • Uses tags to structure content 🏷️

  • Self-closing tags for elements like images 🎞️

  • ❗Four Required tags (boilerplate):
    <!DOCTYPE html>, <html>, <head>, & <body>

HTML Content Tags


HTML Content Tags



  • Building blocks webpage content 🧱

  • Provide structure & meaning to everything inside the <body> 📊

Common

Content Tags



  • Headings (titles & section labels) <h1>-<h6>

  • Paragraphs of text <p>

  • Images <img>

  • Links to other pages or sites <a>

  • Lists <ul>, <ol>, <li>

Other common

content tags:



  • Hyperlinks, Images, Videos, Audio 🔗

  • Tables, Figures 📊

  • Interactive elements such as:
    Buttons, Inputs, Selects 🖱️

  • Divider or Span (containers that group content) 🗃️

  • <!-- Comment --> (notes or reminders) 💬

Semantic HTML


Semantic HTML



  • Uses tags that describe the meaning of the content they hold 🏷️

  • Clarifies role of different parts of the page 💡

  • Improves accessibility ♿
    (screen readers and other assistive technology)



  • Supports SEO 🔍
    (search engine optimization)

  • Makes code easily readable and maintainable for dev teams 👥

Semantic example:



Malala Yousafzai
("muh-LAH-luh YOU-suhf-zye")



You can quote Malala using a <span> or a <q> tag



<span>
  We realize the importance of our voices only
  when we are silenced.
</span>
We realize the importance of our voices only when we are silenced.



<q cite="https://www.goodreads.com/book/malala">
  We realize the importance of our voices only
  when we are silenced.
</q>
We realize the importance of our voices only when we are silenced.

🧠 Meaning matters:



  • <span> a generic container with no meaning

  • <q> semantic tag that tells the browser and assistive tech "this is a quotation"

♿ Accessibility

& clarity:


  • <span> is read as generic text

  • Screen readers announce <q> as a quote


🦾 Best Practice:
Be Semantic!




Semantic Tags


📐 Structure



  • <header>
    Intro or navigational section

  • <nav>
    Navigation links

  • <main>
    Main content



  • <section>
    Thematic grouping of content

  • <article>
    Independent, self-contained content (blog post, news article)

  • <aside>
    Related content not part of main flow (Sidebar, tips)

  • <footer>
    Footer content for a section or the page

📝 Text-Level



  • <h1> - <h6>
    Headings (most to least important)

  • <p>
    Paragraph

  • <q>

  • <cite>

🗣️ Emphasis & Meaning



  • <strong>

  • <mark>

  • <code>
    Inline computer code

  • <pre>

Summary 🧐



  • Meaningful tags describe the role of content 🏷️

  • Improves Accessibility 🦾

  • Enhances SEO 🔍
    (search engine optimization)

  • Makes code easier to read, understand, and maintain 🧑‍🏫

Semantic HTML =
meaningful structure + better usability for ALL users



HTML Best Practices


HTML Standards



  • Case sensitivity 🔤
    (not case sensitive, best practice consistent lowercase)

  • Indent nested elements 📏
    (organized, easy to read)


  • Always close tags 🔚
    (exception= self-closing tags)
    (avoid display issues or broken structure)

  • Meaningful file and folder names 🗂️
    (content easy to find/maintain)

  • Add comments 💬
    (helpful notes/reminders)


  • Validate your code ✅
    (catch errors, ensure markup follows standards)

  • Keep it readable 👀
    (spacing, line breaks)

  • Be semantic 🧩
    (improve structure, accessibility & SEO)

HTML Tag Attributes



Attributes



  • Give extra info/functionality to HTML elements ℹ️

  • Help with behavior of content 📖
    (style, link, label, or control)

  • Some are universal 🌐
    (<id> & <class>)

  • While others are element-specific 🎯
    (<src> & <img>)


  • 🏷️ Set inside the opening tag right after the element name (includes self-closing tags)


  • Written as:


    • attributeName="value here"

HTML Attribute

Categories



  • Standard Attributes: Supported by all HTML elements
    (<id>, <class>)

  • Element-Specific Attributes: Only work with certain tags
    (<href>, <a>)

  • Custom Data Attributes: Created for unique needs
    (<data-*>)

Important Attributes


  • id

  • class

  • href

  • src

  • data-*:

id



  • Supported in all HTML elements

  • Uniquely identifies an element 🪪

  • Used for styling, linking, or scripting specific elements ⚙️

  • ❗ Only use once per page

    • <p id="#hot-toddy"></p>

class



  • Supported in all HTML elements

  • Shared style or behavior to multiple elements 👥

  • Can be reused across your site or page ♻️

  • Best practice: class over id 👌

  • Ideal for CSS or JavaScript targeting 🎯

    • <p class="hot-toddy"></p>

href



  • Used in <a> tags to make clickable links 🖱️

  • Defines the URL that a hyperlink goes to when clicked 🔗

  • Can link to other 🌐 websites or 📎 internal website pages

    • <a href="https://en.wikipedia.org/wiki/Women_in_computing">
    Click here to learn about Women in computing</a>

Click here to learn about Women in computing

src



  • Used in tags like <img>, <video>, <audio> 💾

  • Points to the file path of an image, video, or audio 🗂️

  • Supports relative 📎 (/images/randomImage.jpg)
    or absolute paths 🌐 (https://...)


    • <img src="https://arc.net/images/logo.png" alt="Arc browser logo" height="400vh" />

Relative Path


  • Links internal files 📎
    (images, pages, or stylesheets)

  • Points to file within the same project/directory structure 📁

  • Doesn’t include domain name — just the file’s location relative to the current file 📍


about.html ../images/photo.jpg

Absolute Path


  • Links to external websites or files hosted outside of project 🔗

  • Includes full URL or file location 🌐
    (starting with https:// or http://)

  • Always loads from the same source, no matter where the current file is located 🔄


https://awkwardfamilyphotos.com/category/photos/awkward-hall-of-fame/ awkwardfamilyphotos.com

data



  • Not displayed visually, but used by JavaScript 🤖
    (pulls data from prebuilt database)

  • Gives flexibility without affecting layout or structure 🪄

  • Stores custom info for an element 📇
    (like data-price="12" or data-id="42")

    • <div data-menuItem="3">Hot Toddy</div>

Summary 🧐



  • Attributes add extra information to HTML elements 📝

  • 🏷️ Inside opening tag & appear as name-value pairs (name="value")

  • Some are 🌐 universal (used on most elements) while others are 🧑‍🎨 element-specific or custom

  • 🦾 Improves styling, linking, behavior, and accessibility


Document

Object Model


DOM



  • Turns code into objects browsers can understand & interact with 🤖

  • Tree-like structure browser's build from HTML code 🌳

  • Every HTML element becomes a node in this tree
    (branches & leaves) 🌿

  • Like a map of a page with live updates when things change 🗺️

How DOM

differs from code



  • HTML code is just static text 📝

  • Browser turns it into the DOM 🌳
    (a live, interactive structure)

  • DOM is a representation of the code 🖥️
    (model browsers use to understand & display webpages)



  • HTML stays the same 🔐 unless you change it 🔑

  • DOM can change in real time with JavaScript ⚡

  • You can see and interact with the DOM in browser dev tools 🧪




🔗 Get started with viewing and changing the DOM

Summary 🧐



  • Browser’s internal representation of HTML 🌐

  • Happy little tree of elements 🌳 — each tag becomes a branch of the tree (nodes) 🌿

  • Turns static code into dynamic objects 🤖

  • Changes to HTML update the DOM in real time ⚡

  • You can explore and debug the DOM using browser dev tools 👩‍🔬

Accessibility

Awareness


  • A11y is a numeronym for accessibility ♿
    uses first & last letter with 11 letters in between
    (A + 11 + Y)

  • Accessibility is about making spaces, tools, and experiences usable by everyone, including people with disabilities 🌍

  • Not just ethical — can also be required by law ⚖️

Digital Accessibility



  • Designing websites and apps so that every person can use them effectively 👩‍🦯

  • Ensures content is usable, understandable, and inclusive for a wide range of users — across all devices and abilities 🌐

  • Goal is to create a more inclusive, usable web for all users 🤝

Includes things like:



  • alt text for images 🖼️

  • keyboard navigation ⌨️

  • screen reader support 🧏‍♀️

POUR Principles


What is POUR?



  • Framework from WCAG to make websites more accessible

  • WCAG (Web Content Accessibility Guidelines)




🔗 W3C WCAG 2.0 Guidelines

What Does

POUR Stand For?



  • Perceivable

  • Operable

  • Understandable

  • Robust

Perceivable



  • Content must be available to the senses (sight, hearing, touch) 👩‍🦯

  • Use alt text for images, captions for video, screen reader cues, & clear color contrast 🦻

Operable



  • All users must be able to interact with the interface, regardless of ability 👩‍🦼

  • Ensure all functionality works with a keyboard ⌨️
    (not just a mouse 🖱️)

  • Avoid motion triggers (like parallax scrolling) that can cause issues for some users 🚫

Understandable



  • Information and navigation should be clear, consistent, & predictable 🧠

  • Use simple language, readable fonts, and helpful error messages ⚠️

Robust



  • Content should work well with current and future assistive technologies 🚀

  • Use clean, semantic HTML that supports screen readers and other assistive tools 🧼

Accessibility

Best Practices


Accessibility

Best Practices



  • Add alt text for all images 🖼️ → Helps screen readers describe visual content to users who are blind or have low vision

  • Ensure keyboard navigation ⌨️ → All interactive elements should be accessible without a mouse

  • Use sufficient color contrast 🌈 → Makes text easier to read for users with low vision or color blindness


  • Provide clear labels & instructions 💬 → Improves comprehension and usability, especially for assistive tech users

  • Include error messages and feedback ⚠️ → Let users know what went wrong and how to fix it

  • Keep navigation consistent 🔄 → Predictable structure helps reduce confusion

  • Use captions or transcripts for audio/video 🔊 → Supports Deaf and hard-of-hearing users (and helps everyone in noisy or quiet environments)


  • Use semantic HTML 🧱 → Helps assistive tech understand your content's meaning and structure

  • Test with accessibility tools 🧪 → Use Lighthouse, axe DevTools, or screen readers to identify and fix issues

  • Design for all users 👥 → Think beyond “average” — include users with diverse abilities, needs, and technologies

Summary 🧐



  • A11y helps create better experiences for all users (inclusive, ethical, often legally required) ⚖️

  • Follow the POUR principles (Perceivable, Operable, Understandable, Robust) 🦾

  • Designing with accessibility in mind benefits everyone (users with temporary impairments, mobile devices, or slow connections) 🌍

  • Follow best practices!!

What is

Architecture?



  • The design and structure behind how something is built or organized 🏗️

  • Provides a blueprint or plan to ensure things work well, stay organized, and can grow over time 📈

Front-End

Architecture



  • The organizational structure of your project — how your code and assets are arranged Organized 🗂️

  • Clear architecture makes your code easier to read, debug, and maintain — especially when working with a team 👥

Why Is It Important?


"Easy" to write code,


Hard to write maintainable code!



Architecture

May Vary based on:


  • Project requirements 📌

  • Device requirements 📱
    (Mobile, tablet, or desktop)

  • Frameworks used ⚙️
    (Some req their own architecture)

  • New features are added ✨

  • Size of your team scales 📈

What Does

This Look Like?



Mostly conceptual
