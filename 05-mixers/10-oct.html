SPA Introduction

What is a SPA?


Single Page Application


Single index.html page


Dynamically updates the page
based on user interaction

(without reloading the entire page!)


Better User Experience

'conventional' website



Different HTML pages (files)


Each URL serves up a different page


Files might share/import resources

...but that's as good as it gets


Single Page Application


You probably use them everyday

...but you might not even know it!


Every URL updates a portion of the page. Behaves more like an app on your PC or phone


Single Page Application


Obvious: Pages with scrollable sections


Less obvious: The URL changes but it's still served from a single index.html file

AJAX & Single Page Apps


Asynchronous JavaScript and JSON (or XML)


Data is exchanged between browser and server in the background

Requires JavaScript to be enabled in the browser (like SPAs in general)

AJAX allows our application to retain a single page while making calls to the server

Synchronous vs Asynchronous


Synchronous: Code is executed in order and waits until previous action completes before moving to next action


Asynchronous: Code is executed, but doesn't wait for previous actions to complete

Single Page Application Cons



Anyone browsing with JavaScript disabled will not get all the functionality


Single Page Application Cons


Search Engine Optimization (SEO)


A data or JavaScript error can mean you have no content for bots to crawl!


Direct bots to prerendered content
Implement cache with prerender.io

Time to get Angular


Choosing a framework or library

Why did we choose Angular?


CLI helps scaffold our project quickly


Angular is very opinionated
so this reduces architectural decisions


Good documentation


Angular uses TypeScript


Made by Microsoft


.ts file extensions instead of .js


Compiles into JavaScript


Can write JavaScript in a TypeScript file
(but not the other way around!)

Why Use TypeScript?


JavaScript is loosely typed
a variable's value is flexible


var drink = "Tom Collins";
drink = 3;

Why Use TypeScript?


TypeScript provides static "types" (number, string, boolean, etc) for JavaScript

var drink = "Mojito";
drink = true; //ERROR!
drink="Long Island";

A variable's type is enforced, but not the value.

Angular CLI

(Command Line Interface)


Check out the documentation here


CLI does things the "right" way
according to Angular style guide


Install via npm npm install -g @angular/cli

What can we do with the CLI?


Create a new project ng new PROJECT-NAME-HERE --routing


Serve your project
(so we can see it in the browser) ng serve

What can we do with the CLI?


Scaffold components, services, directives, etc.


Component ng generate component MY-NEW-COMPONENT ng g component MY-NEW-COMPONENT

Service ng g service MY-NEW-SERVICE

CLI Scaffolding

Build: create dist directory w/ build artifacts ng build


Install angular-cli-ghpages (one time only) npm install -g angular-cli-ghpages

What can we do with the CLI?


Build to your GitHub repo ng build --prod --base-href "https://yourusername.github.io/reponame/"


And deploy! angular-cli-ghpages

Modules

related sections of functionality

Components


Every app has at least one root

Application is a tree of components

Each component contains:
template, data, behavior

Components

<app-quiz></app-quiz> @Component({
  selector: 'app-quiz',
  templateUrl: './quiz.component.html',
  styleUrls: ['./quiz.component.css']
})

Lifecycle of a Component


"Hooks" are events that signal a change in the component's lifecycle


Main lifecycle hook is ngOnInit


- Called once shortly after Angular creates the component

- Commonly used for fetching the initial data needed for the component

Read more here

Routing


Tells your application what to display for a URL


Main homepage route is '/' and is created by Angular CLI


...but we have to add the other routes

Other route examples:
'/about' or '/profile/:username'


View more on routing here

Templates are HTML


Tags we can't use: <script><html><body><base>


Tell your TypeScript @Component code that your html template lives in a separate HTML file: templateUrl: './header.component.html'

Interpolation


An expression that Angular evaluates and converts to a string.

Surrounded by double curly braces: {{ }}


{{animal.name}} {{3 + 5}} <p>My age is {{getMyAge() - 5}}</p>
<!-- My age is 21 -->

Data Binding


Pass data around your application

The way data is bound depends on syntax


  • Component to the view using {{}}

  • View to component ()



Data Binding


{{ }} data is sent to the view from the component


<p>My dog's name is {{user.dogsName}}</p>

This is typically where you'd use text in HTML

Data Binding


( ) is a value taken from the view and passed to the component (user interaction w/ the page)


<button (click)="delete()">Delete</button>

This is typically seen on attributes
in template elements

Adding data to a component


  • Start with a model: a structured way of representing an object

  • Import the model to the component

  • Setup or fetch initial data in OnInit, one of the component's lifecycle hooks

  • Finally, write HTML for the template
