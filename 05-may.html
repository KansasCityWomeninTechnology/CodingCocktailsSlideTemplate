Introduction

What is it?



Architecture is conceptual...
how files are organized



Building a house requires good architecture...
the same applies to apps/sites

What is it?


A toolshed is not like a baseball stadium...

different apps/sites require different structures

Why is it important?


Promotes code reuse
Don't Repeat Yourself!

Why is it important?


Single Responsibility Principle:
chunks of code that do one thing


Single Responsibility Principle:
chunks of code that do one thing


Why is it important?

These chunks are called modules/components


Easier to maintain over time
(happy devs, happy clients, cost savings)

Concepts & Preferences


Architecture can vary based on:


  • Project requirements
  • Technologies chosen
  • Team preferences
    • imgs vs images
    • app/about/css vs app/css/about
    • dashed-names vs underscore_names

The Structure


Ahh... that's more like it

App Directory


Houses your application's code


/app
/assets
/dist
index.html - Most important file
LICENSE.txt
...

Assets Directory



/assets
/images
/styles
CSS or Sass content
/scripts
JavaScript functionality
/fonts
Only if serving font from your server
/views (ie Handlebars)
Full or partial markup

Dist Directory

  • HTML/CSS/JS that gets served to the browser
  • Minified & optimized code
    • myVeryLongFunctionName => r
    • Removes extra spaces and lines
    • Reduces readability
    • Improves performance
  • Created by the build process
    • Version Control
      The dist directory should not be tracked

Scaffolding with Yeoman

Yeoman is here to help!

What is Yeoman?


Scaffolding tool that sets up your project's file stucture



Language agnostic
(HTML/JS/CSS, Python, C#, Java, PHP, etc)


How to use Yeoman?


Varies by language & requirements


npm install -g yo


Type 'yo' in your Terminal (Cmder or iTerm2)

What is a Generator?

A Yeoman plugin that defines an architecture template

7000+ generators available

Install a generator with npm install -g generator-whatever

Can't find a solution for your app?
Create your own!

A Yeoman project using Webapp Generator


What Does Fountain-Webapp Give Us?


  • Basic File Structure
  • CSS Autoprefixing
  • Compiles SASS
  • Live Reload of project
  • Lints scripts
  • Image Optimization
  • Creates a 'dist' folder on command
  • And more...

This Webapp uses Gulp

  • Start local server and watch for changes gulp serve

  • Compiles and optimizes files to make deployment-ready gulp

  • Preview production files gulp serve:dist

...but npm is the starting point

  • npm run ____ npm run serve

Need help?

  • With Yeoman yo --help

  • Get details on a generator yo webapp --help

  • Troubleshoot yo doctor
