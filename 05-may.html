Introduction

What is it?


Architecture is conceptual...
how files are organized


Building a house requires good architecture...
the same applies to apps/sites


A toolshed is not like a baseball stadium...
different apps/sites require different structures

Why is it important?


Promotes code reuse
Don't Repeat Yourself!


Single Responsibility Principle:
chunks of code that do one thing


These chunks are called modules/components


Easier to maintain over time
(happy devs, happy clients, cost savings)

Concepts & Preferences


Architecture can vary based on...

  • Project requirements (responsiveness, flexibility, release/sales strategy)
  • Technologies chosen (SASS/Less, Framework guides & best practices)
  • Team [Lead] preferences ('imgs' vs 'images', 'app/about/css' vs 'app/css/about', dashes vs underscores)

The Structure

App Directory

  • Houses your applicationâ€™s code
    • Index.html - most important file
    • Assets
      • Images - self explanatory
      • Styles - css, scss, fonts
      • Scripts - javascript (or other language) functionality
      • Fonts - only if serving up the font from your server
      • Views - full or partial markup (ie Handlebars)
    • Dist
      • Minified & optimized, for serving to the browser
      • Created by the build process
      • As a build artifact, 'dist' should not be tracked by version control

Dist

  • Short for "distribution"
  • Production-ready
  • Files are made to load quickly - they're minified
    • Removes spaces, replaces human-created function names to shorter names. getDrink() -> x()
    • Not very human readable, but are read quickly by the browser for better performance
  • Can be manually created, but usually developers use a tool like Gulp to automate the process.

In dist, you have...

  • index.html (dist/index.html)
  • Assets
  • Views

What is Yeoman?

Scaffolding tool that sets up your project's file stucture

Language agnostic (HTML/JS/CSS, Python, C#, Java, PHP, etc

Scaffolding is the structure of your project. Differs depending on languages used and requirements of project.

Install via NPM 

npm install -g yo

Use it with 'yo' in the CLI

What is a Generator?

A Yeoman plugin that defines an architecture template

4000+ generators available

Install a generator with `npm install -g generator-whatever`

Can't find a solution for your app? Create your own!

A Yeoman project using Webapp Generator

What Does Webapp Give Us?

  • Basic File Structure
  • CSS Autoprefixing
  • Compiles SASS
  • Live Reload of project
  • Lints scripts
  • Image Optimization
  • Creates a 'dist' folder on command
  • And more...

Webapp uses Gulp

  • Start local server and watch for changes
    • gulp serve
  • Compiles and optimizes files to make deployment-ready
    • gulp
  • Preview production files
    • gulp serve:dist

Need help?

  • With Yeoman
    • yo --help
  • Get details on a generator
    • yo webapp --help
  • Troubleshoot
    • yo doctor
