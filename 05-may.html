Introduction

What is it?

Where files & assets go (how they’re organized)

Good architecture is a fundamental step in building a house...

 the same applies to applications/websites!


A toolshed is not the same as a baseball stadium...

 different applications/websites can require different structure!

Why is it important?

Promotes code reuse

Don't Repeat Yourself!

Single Responsibility Principle: chunks of code that do one thing

These chunks are called modules, components, widgets, etc


Easier to maintain over time (happy devs, happy clients, cost savings)

Concepts & Preferences

Architecture can vary based on...

Project requirements (responsiveness, flexibility, release/sales strategy)

Technologies chosen (SASS/Less, Framework guides & best practices)

Team [Lead] preferences ('imgs' vs 'images', 'app/about/css' vs 'app/css/about', dashes vs underscores)

The Structure

App Directory

  • Houses your application’s html/css/js code.
    • Index.html
    • Assets
    • Dist
  • Unminified, unoptimized, for development

Index.html

  • The most important page for your site - the Home Page
  • What gets served (without /index.html appended to the end of the url).
  • When you view www.amazon.com, you're viewing amazon.com's index.html file, for example.

Assets

  • Images
  • Styles
  • Scripts
  • Fonts
  • Views

Images

  • .png or .jpg files
  • Can be called imgs/ or images/
  • Can organize it even further into images/apps or images/mobile

Styles

  • .css files
  • Other names: `css/`, `scss/`, `style/`, etc.
  • Can organize further into variables, mixins, app name

Scripts

  • .js files
  • Other names: `js/`, `javascript/`, `script/`, etc.
  • Can have subfolders for "libraries", like jQuery, Bootstrap, FontAwesome

Fonts

  • .otf, .ttf files
  • May not need this, Google Fonts is so popular and you don't have to host it yourself.

Views

  • Partial .html files
  • Could be called templates, reusable pieces of code (nav, footer)

Dist

  • Short for "distribution"
  • Production-ready
  • Files are made to load quickly - they're minified
    • Removes spaces, replaces human-created function names to shorter names. getDrink() -> x()
    • Not very human readable, but are read quickly by the browser for better performance
  • Can be manually created, but usually developers use a tool like Gulp to automate the process.

In dist, you have...

  • index.html (dist/index.html)
  • Assets
  • Views
  • Project Requirements
    • Responsiveness - Supporting mobile & desktop? Other devices?
  • Tools / Build Process
    • Preprocessors (like Sass/Less for CSS) - Use scss folder instead of css
    • Frameworks - You’ll want a file structure to complement that framework - i.e. Wordpress has its own style guide and rules to follow

Setting up your project wearing you out?

Yeoman is here to help!

What is Yeoman?

  • Tool that sets up your project's file stucture
  • Can be used with a variety of languages using generators to create scaffolding
  • Scaffolding is the structure of your project. Differs depending on languages used and requirements of project.
  • Install via NPM 
    npm install -g yo
  • Give commands to Yeoman using Yo in the CLI

Setup Yeoman project using Fountain Webapp Generator

Fountain Webapp Features

  • Creates file structure for project
  • CSS autoprefixing
  • Compiles SASS
  • Lints scripts
  • Live Reload of project
  • Image Optimization
  • And much more!

Helpful commands

  • To turn on server and view project
    • npm run serve
  • Compiles and optomizes files to make production ready
    • npm run build
  • Preview production files
    • npm run serve:dist
  • For help using Yeoman
    • yo help
  • For help using generator
    • yo [generator name] --help
  • Running into errors with yo commands? 
    • yo doctor
