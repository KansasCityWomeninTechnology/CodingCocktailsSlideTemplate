Functions (aka Methods)


  • A function performs a task

  • Parameters: the Mad Libs Concept

  • It may or may not "return" an obvious value once it is finished

Functions in the Wild

Parts of a Function


var functionName = function (optional, parameters, here) { //function logic here };

Functions in the Wild

Name your function ('declare a variable')

var orderDrink = function(drinkName)

Read as: The variable "orderDrink" is a function with one parameter, called "drinkName"

Functions in the Wild


Inside the curly braces, is the task that is performed:

var orderDrink = function(drinkName) { return 'I would like a ' + drinkName + ' please.';}

When you run the code, you decide what the value of the parameter is and insert it, Mad Libs style!

Functions in the Wild

finally, "call" the function by telling it what to use for the parameter:
