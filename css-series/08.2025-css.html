CSS Basics



  • Language that tells web browsers how to display HTML elements 🖥️✨

  • Reusable rules across a website 🔁

  • Used in every modern website 🌐

  • Easy to start, powerful as you grow 🚀

CSS control?



  • Webpage Appearance 🎨
    (colors, fonts, spacing)

  • Content Layout 🏗️
    (alignment, borders, boxes, positioning)

Appearance

Layout



  • Control how elements are arranged on the page 🏗️

  • Determine number of columns, element alignment (horizontally or vertically), image sizes & image captions 📏

How to

Write CSS



  • Inline 🏷️
    (inside the tag)

  • Internal 🗂️
    (in a <style> tag inside the <head> of an HTML file)

  • External 🌐
    (.css file linked in the <head> of an HTML file)

Inline

<p style="color: #c7bc75;">CSS so smooth, it's got a peach fuzz finish! 🍑</p>



  • Quick and easy for one-time changes ⚡

  • Can clutter HTML code 🧹

  • Not reusable 🙅‍♀️

Internal

<head>
 <style>
   p {
     color: #c7bc75;
    }
 </style>
<head>

  • Keeps styles in one place within the HTML 📄

  • Good for small projects or prototypes ⚗️

  • Can’t share styles with other files 🙅‍♀️

External

<link rel="stylesheet" href="styles.css" /> p {
  color: #c7bc75;
}
  • Linked in the <head> of HTML using a <link> tag 🔗

  • Clean & organized structure 🧼

  • Reusable across multiple pages & projects 🔄

  • Best practice for most websites ✅

Linking

Attributes


<link rel="stylesheet" href="styles.css" />
  • rel:
    describes the relationship between the current & linked file to the browser

  • href:
    specifies the file's path (relative or absolute)


  • Relative:
    refers to a page in the same folder as the current HTML file 📁
    • <link rel="stylesheet" href="styles.css" />

  • Absolute:
    refers to a path outside of the project & includes the entire URL 🌐
    • <link rel="stylesheet" href="https://example.com/styles.css" />

Combinators



  • Combine selectors to define relationships between elements 🤝

  • Target elements based on position in the DOM 🌳

  • Create more precise & efficient style rules 🪄

  • Reduce extra classes or IDs in your HTML 🗑️

  • Four main types 👨‍👩‍👧‍👦:
    descendant, child, adjacent sibling, general sibling

Descendant

Combinator

    color: lime;

  • Great for broad styling inside containers 🗃️

    Values:

    Units of Measurement



    • CSS units describe length, size, spacing, etc. 📏
      (relative or absolute)

    • Absolute units are fixed sizes, always the same regardless of context 📐

    • Relative units depend on something else (like the parent or viewport) & adapt responsively 🌱

    Absolute



    • Fixed size — stay the same regardless of screen or zoom 🔒

    • pixels [px] (Most common)

    • [cm], [mm], [in] (Mostly print-focused)

    • Points [pt], Picas [pc](Print typography)

    Pixels [px]

    h1 {
      font-size: 32px;
      margin-bottom: 16px;
    }
    • Most common absolute unit

    • 1px = 1 dot on the screen (varies by device)

    • Can be used for:
      width, height, margin, padding, font-size

    Relative



    • Scale based on an element or viewport — flexible & responsive 🔄

    • Percent [%] - Relative to parent element’s size

    • rem / em - Relative to the font size of the current:
      root = [rem] element =[em]

    • vw / vh - Percentage of browser window
      (viewport width & height)

    Percent [%]

    .container {
      width: 80%; /* container takes 80% of its parent’s width */
      padding: 5%; /* padding is 5% of container’s width */
      background-color: lightblue;
    }
    • Relative to parent element

    • Can be used for:
      width, height, margin, padding, font-size

    rem / em

    /* Assuming root font-size is 16px */
    /* REM: Root | Em: Parent element */

    h1 {
      font-size: 2rem; /* 2 * 16px = 32px */
    }

    p {
      font-size: 1.5em; /* 1.5 * parent font-size */
    }
    • If root element is 16px, 1rem for an element = 16px

    • Can be used for:
      width, height, margin, padding, font-size

    vh / vw

    /* Goal: Make a full-screen hero section * /

    .hero {
      height: 100vh; /* 100% of viewport height */
      width: 100vw; /* 100% of viewport width */
      background-color: lightcoral;
    }
    • Relative to viewport height / width (% of viewport)

    • Can be used for:
      width, height, margin, padding, font-size

    Color



    • Bring webpages to life with color 🌈

    Color Formats



    • Named Colors 🎨:
      Easy-to-remember color names (red, blue, teal)

    • Hexadecimal (Hex) #️⃣:
      Starts with # plus 3 or 6 hex digits (0–9, A–F)
      #ff6347 or shorthand #f63

    • RGB 🎛️:
      Defines colors by mixing red, green, & blue values from 0–255
      rgb(255, 99, 71)


    • RGBA 🌫️:
      Like RGB but adds alpha for transparency
      rgba(255, 99, 71, 0.7)

    • HSL 🌈:
      Uses hue (color type), saturation (color intensity), & lightness (brightness)
      hsl(9, 100%, 64%)

    • HSLA 🌫️:
      HSL with alpha transparency
      hsla(9, 100%, 64%, 0.7)

    RGB

    /* Goal: Make all <h1> elements red * /

    h1 {
      color: rgb(255, 0, 0);
    }
    • 3-digit RGB code

    • Each digit represents 256 values (0-255)

    • Each set of digits represents the intensity of a color (red, green, blue)

    Hexidecimal Numbers

    /* Goal: Make all <h1> elements red * /

    h1 {
      color: #ff0000;
    }
    • Hexadecimal is a base 16 number system

    • Each digit represents 16 values (0-9, A-F)

    • Each pair of digits represents a color (red, green, blue)

    🤔 Quick Note:

    About Hex 🧙‍♀️



    • Hex shorthand uses 3 digits instead of 6 to save space ✂️

    • Each digit doubles to form the full 6-digit code 🔄
      (#f63 = #ff6633)

    • Works only when hex pairs are identical 👯‍♀️
      (ff, 66, 33)

    • Makes CSS cleaner & easier to read 🧼

    The Cascade



    • Process of combining multiple stylesheets and resolving conflicts between different CSS rules and declarations

    • Order of CSS rules are important - there's an hierarchy to the cascade that determines which rules are applied

    Source Order



    • Later rules win over earlier rules (of the same specificity)

    Specificity



    • Algorithm that browsers use to decide which rule to apply when multiple selectors target the same element

    • More specific rules win over less specific rules

    Order of Specificity

    (lowest to highest)

    1. Universal selector (*)

    2. Element selectors (p, h1)

    3. Classes (.myClass)

    4. IDs (#myId)

    5. Combining selectors to make rules more specific

    6. !important (WARNING: Overrides all other rules. Bad practice.)

    Box Model

    • Spacing inside the box, outside the box, and the cardboard itself takes up space

    • Padding: the space between the pizza & the box

    • Border: the cardboard box

    • Margin: the space outside the pizza box (and the space between the pizza box and another box)

    Advantages of Flexbox and CSS Grid



    • Handles complex layout needs

    • Manages wrapping and flexibility across various display sizes (phones, smart displays, tablets, big tv's)

    • Provides maintainable code without needing workarounds or hacks

    • Now the standard way of adding layout

    1-dimensional Layout



    • Use CSS Flexbox

    • Elements lay out next to each other in either row or column (but not both)

    • Elements can wrap to multiple rows or columns

    • Example:
      https://codepen.io/chriscoyier/full/PoPwPbE

    2-dimensional Layout



    • Use CSS Grid

    • Elements lay out horizontally and vertically from each other in both rows and columns

    • Used more often

    • Example:
      https://codepen.io/chriscoyier/full/ZEpPWKO

    Flexbox & Grid



    • Flexbox is 1-dimensional

    • Grid is 2-dimensional

    • They are not an either/or layout system

    • You can use BOTH when styling

    CSS Grid Nom Nom Gallery



    https://codepen.io/ramenhog/full/MpORPa

    • Shows grid with defined height (1st row) and dynamic heights (3rd row and on) in masonry pattern

    • Responsive

    • Type of layout is very painful without grid

    • Flexbox handles the responsiveness

    How to Use a Layout System



    • HTML: Create a parent container with a class to style

    • CSS: Define the styling to add grid and/or flexbox

    HTML



    • Common parent containers:
      main, header, footer, form, section, article, div


    <section class="container">
      <!-- Child elements here -->
    </section>

    CSS



    • In the styles for the container, add the following:


    display: grid;
    display: flex;
    .container {
      display: grid;
    }

    CSS Frameworks



    • Collection of pre-written CSS code that you can use to style your website with pre-defined styles

    • Useful for quickly styling a website

    • Some CSS frameworks allow for customization

    • Many development teams use CSS frameworks instead of writing their own CSS

    Why Use CSS Framework?



    • Allows developers to focus on application features rather than styles

    • Styling sites becomes much faster

    • CSS frameworks are designed to be responsive

    • Downside:
      Learning curve of learning the styles of the framework

    How Does This Work?



    • Import the framework into your site:

      • <!doctype html>
      <html>
      <head>
      <meta charset="UTF-8">
      <meta name="viewport" content="width=device-width, initial-scale=1.0">
      <script src="https://cdn.tailwindcss.com"></script>
      </head>
      </html>

    How Does This Work?



    • Use the styles provided by the framework:

      • <h1>Hello world!</h1>

      <h1 class="text-3xl font-bold underline text-pink-500 background-pink-50">
          Hello world!
      </h1>
