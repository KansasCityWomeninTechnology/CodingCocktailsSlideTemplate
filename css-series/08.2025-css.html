CSS Basics



  • Language that tells web browsers how to display HTML elements 🖥️✨

  • Reusable rules across a website 🔁

  • Used in every modern website 🌐

  • Easy to start, powerful as you grow 🚀

What does

CSS control?



  • Webpage Appearance 🎨
    (colors, fonts, spacing)

  • Content Layout 🏗️
    (alignment, borders, boxes, positioning)

Appearance



  • Define styling rules for elements on a webpage ✍️

  • Similar to font styling tools in MS Word 📎
    (color, size, spacing)


Layout



  • Control how elements are arranged on the page 🏗️
    (MS Word: organize text & images)

  • Determine number of columns, element alignment (horizontally or vertically), image sizes & image captions 📏

Basically...



  • It makes webpages look pretty!

  • Syntax:
    .css

  • Written as property value pairs:
    selector { property: value; }

  • Stored in a dedicated "styles" folder 📂

  • Each rule targets an element to determine what it should look like 🎯

How to

Write CSS



  • Inline 🏷️
    (inside the tag)

  • Internal 🗂️
    (in a <style> tag inside the <head> of an HTML file)

  • External 🌐
    (.css file linked in the <head> of an HTML file)

Inline

<p style="color: #c7bc75;">CSS so smooth, it's got a peach fuzz finish! 🍑</p>


  • Quick and easy for one-time changes ⚡

  • Can clutter HTML code 🧹

  • Not reusable 🙅‍♀️

Internal

<head>
 <style>
   p {
     color: #c7bc75;
    }
 </style>
<head>
  • Keeps styles in one place within the HTML 📄

  • Good for small projects or prototypes ⚗️

  • Can’t share styles with other files 🙅‍♀️

External

<link rel="stylesheet" href="styles.css" /> p {
  color: #c7bc75;
}
  • Linked in the <head> of HTML using a <link> tag 🔗

  • Clean & organized structure 🧼

  • Reusable across multiple pages & projects 🔄

  • Best practice for most websites ✅

Linking

Attributes


<link rel="stylesheet" href="styles.css" />
  • rel:
    describes the relationship between the current & linked file to the browser

  • href:
    specifies the file's path (relative or absolute)


  • Relative:
    refers to a page in the same folder as the current HTML file 📁
    • <link rel="stylesheet" href="styles.css" />

  • Absolute:
    refers to a path outside of the project & includes the entire URL 🌐
    • <link rel="stylesheet" href="https://example.com/styles.css" />

Combinators



  • Combine selectors to define relationships between elements 🤝

  • Target elements based on position in the DOM 🌳

  • Create more precise & efficient style rules 🪄

  • Reduce extra classes or IDs in your HTML 🗑️

  • Four main types 👨‍👩‍👧‍👦:
    descendant, child, adjacent sibling, general sibling

Descendant

Combinator

/* Goal: Style all <em> elements inside a <ul> lime green * /

ul em {
    color: lime;
}
  • Uses space between two selectors 📏

  • Selects any element nested inside another (any depth) 🗂️

  • Great for broad styling inside containers 🗃️

Child

Combinator

/* Goal: Style only direct children of a <ul> lime green * /

li > em {
    color: lime;
}

  • Uses > between two selectors

    • <!-- This path does not exist -->
    ul > em {

Adjacent

sibling combinator

/* Goal: Style the first <li> after any <ul> lime green * /

  • Uses + between two selectors

    Pairs

    Values:

    Units of Measurement



    • CSS units describe length, size, spacing, etc. 📏
      (relative or absolute)

    • Absolute units are fixed sizes, always the same regardless of context 📐

    • Relative units depend on something else (like the parent or viewport) & adapt responsively 🌱

    Absolute



    • Fixed size — stay the same regardless of screen or zoom 🔒

    • pixels [px] (Most common)

    • [cm], [mm], [in] (Mostly print-focused)

    • Points [pt], Picas [pc](Print typography)

    Pixels [px]

    h1 {
      font-size: 32px;
      margin-bottom: 16px;
    }
    • Most common absolute unit

    • 1px = 1 dot on the screen (varies by device)

    • Can be used for:
      width, height, margin, padding, font-size

    Relative



    • Scale based on an element or viewport — flexible & responsive 🔄

    • Percent [%] - Relative to parent element’s size

    • rem / em - Relative to the font size of the current:
      root = [rem] element =[em]

    • vw / vh - Percentage of browser window
      (viewport width & height)

    Percent [%]

    .container {
      width: 80%; /* container takes 80% of its parent’s width */
      padding: 5%; /* padding is 5% of container’s width */
      background-color: lightblue;
    }
    • Relative to parent element

    • Can be used for:
      width, height, margin, padding, font-size

    rem / em

    /* Assuming root font-size is 16px */
    /* REM: Root | Em: Parent element */

    h1 {
      font-size: 2rem; /* 2 * 16px = 32px */
    }

    p {
      font-size: 1.5em; /* 1.5 * parent font-size */
    }
    • If root element is 16px, 1rem for an element = 16px

    • Can be used for:
      width, height, margin, padding, font-size

    vh / vw

    /* Goal: Make a full-screen hero section * /

    .hero {
      height: 100vh; /* 100% of viewport height */
      width: 100vw; /* 100% of viewport width */
      background-color: lightcoral;
    }
    • Relative to viewport height / width (% of viewport)

    • Can be used for:
      width, height, margin, padding, font-size

    A11y Reminder 🦾



    • Use legible fonts with clear shapes & spacing 🔤

    • Avoid very small font sizes — keep text readable 🔍
      (14px or larger)

    • Avoid fixed sizes that can break layouts on different devices 📱💻

    • Consider user zoom preferences
      & screen readers 🧑‍🦯

    Color



    • Bring webpages to life with color 🌈
      (text, backgrounds, borders, & more!)

    • CSS offers many ways to define colors 🎨
      (named colors, hex codes, RGB, HSL, & transparency)

    • Colors set the mood, improve readability, & make your designs pop! 💥

    A11y Reminder 🦾



    • Use sufficient contrast for readability 🌓

    • Don’t use color alone to convey meaning — add labels or icons 🏷️

    • Test colors with accessibility tools to support all users 🧪

    • Design for color blindness & visual impairments 👓

    Color Formats


    • Named Colors 🎨:
      Easy-to-remember color names
      black, blue, red

    • Hexadecimal (Hex) #️⃣:
      Starts with # plus 3 or 6 hex digits (0–9, A–F)
      #ff6347 or shorthand #f63

    • RGB 🎛️:
      Defines colors by mixing red, green, & blue values from 0–255
      rgb(255, 99, 71)


    • RGBA 🌫️:
      Like RGB but adds alpha for transparency
      rgba(255, 99, 71, 0.7)

    • HSL 🌈:
      Uses hue (color type), saturation (color intensity), & lightness (brightness)
      hsl(9, 100%, 64%)

    • HSLA 🌫️:
      HSL with alpha transparency
      hsla(9, 100%, 64%, 0.7)

    RGB

    /* Goal: Make all <h1> elements red * /

    h1 {
      color: rgb(255, 0, 0);
    }
    • 3-digit RGB code

    • Each digit represents 256 values (0-255)

    • Each set of digits represents the intensity of a color (red, green, blue)

    Hexidecimal Numbers

    /* Goal: Make all <h1> elements red * /

    h1 {
      color: #ff0000;
    }
    • Hexadecimal is a base 16 number system

    • Each digit represents 16 values (0-9, A-F)

    • Each pair of digits represents a color (red, green, blue)

    🤔 Quick Note:

    About Hex 🧙‍♀️



    • Hex shorthand uses 3 digits instead of 6 to save space ✂️

    • Each digit doubles to form the full 6-digit code 🔄
      (#f63 = #ff6633)

    • Works only when hex pairs are identical 👯‍♀️
      (ff, 66, 33)

    • Makes CSS cleaner & easier to read 🧼

    The Cascade

    Flow of Styles 🌊


    • CSS rules apply in sequence 🔢

    • Specificity decides — more precise selectors win 🥇

    • Last rule wins — if specificity ties, the later rule applies 🔚

    • !important overrides all rules ‼️

    • Best practice: Use !important sparingly 🚨
      (last resort, never default 🙅‍♀️)

    Source

    Order


    • Rules are read top to bottom 📜

    • Later rules override earlier rules 🔄
      (if selectors have same specificity)

    • Group related styles 📑
      (reduce accidental conflicts)

    • Keep CSS organized so overrides are intentional 🗂

    Specificity



    • Browser’s algorithm to decide which CSS rule applies when multiple selectors target the same element 🤖

    • Calculated based on selector types 🧮:
      inline styles > IDs >

    • Selector with highest specificity takes priority 🥇

    • If specificity ties, the rule that appears last in the CSS wins 🤜🤛

    Order

    of Specificity

    (lowest to highest)



    • Universal selector (*)

    • Element selectors (p, h1)

    • Classes (.myClass)


    • IDs (#myId)

    • Combining selectors that make rules more specific

    • Last Resort 🪳:
      !important
      ( Overrides all other rules‼️)
    <h1 class="gif" id="ryan-gosling">Hey Girl!</h1> h1 {
        color: red;
    }
    .gif {
        color: blue;
    }
    #ryan-gosling {

    Inheritance



    • Similar to family traditions 👩‍👧‍👧

    • Styles get passed down from parent elements to children elements 👩‍💼➡️👧

    • Style elements such as <body> or <h1> first 🖌️

    Common

    Layout Systems


    • Flexbox (horizontal/vertical alignment)🟦🟦🟦
      one-dimensional layouts

    • CSS Grid (rows & columns) ▭▭▭
      two-dimensional layouts for complex designs

    • Multi-Column Layouts 📰
      Flowing content into multiple columns, like a newspaper

      Absolute, relative, fixed

    • Float Layouts 🫧
      Old school! Still useful for specific legacy cases

    CSS Flexbox

    1-dimensional Layout


    • Elements lay out next to each 🟦🟦🟦
      (row or column, NOT both 🙅‍♀️)

    • Can wrap to multiple rows or columns📊

    • 🔗 Click here to view an example

    CSS Grid

    2-dimensional Layout



    • Elements lay out horizontally & vertically from each other in both rows & columns 🧱

    • Most common 🤩

    • 🔗 Click here to view an example

    Combining

    Flexbox & Grid


    • Combine Flexbox & Grid to build flexible, responsive layouts that handle both large page structure & smaller component alignment 🤹‍♀️✨

    • Grid for overall page layout 📊
      (rows & columns)

    • Flexbox for arranging items inside grid areas 🔳🔳🔳
      (navigation menus, buttons)

    How to

    Use a Layout System


    • Choose based on your design needs 🎯
      (Flexbox, Grid)

    • Define the container & apply the layout display property 📦
      (display: flex, display: grid)

    • Set up rows, columns, & gaps to organize content visually 🧱


    • Use alignment & spacing properties to control positioning & flow 📐

    • Test on different screen sizes to ensure a good user experience 📱💻🖥️

    • Combine layout techniques as needed for complex designs 🔄

    HTML

    /* Goal: Create a container for the layout * /

    <div class="container">
        <h1>Heading</h1>
        <p>Paragraph text</p>
        <img src="image.jpg" alt="Description">
    </div>

    Common parent containers:


    • main, header, footer, form, section, article, div

    CSS

    /* Goal: Style the container for the layout * /

    .container {
        display: flex;
        flex-direction: column;
        align-items: center;
        justify-content: center;
        width: 80%;
        margin: 0 auto;
    }
    • Helps arrange & style common containers efficiently, creating flexible & clean designs 📐✨
      (<div>, <section>)

    Summary 🧐



    • Every HTML element is a box with content, padding, border, & margin 📦🍕

    • Box model controls element size, spacing, & layout consistency 🎯

    • Margin, padding, & border adjust space inside/outside for clean design 🧼

    • Layout systems like Flexbox & Grid create responsive, adaptable pages 📱💻🖥️

    CSS Frameworks



    CSS Frameworks



    • Pre-built collections of CSS code that speed up styling websites 📚⚡

    • Provide ready-made components like buttons, grids, & forms 🔲

    • Help maintain consistency and save development time ⏳

    • Popular frameworks 💻🔥:

    Why Use

    CSS Frameworks?


    • Speeds development by providing reusable styles & components ⚡

    • Ensures consistency across the entire site ✅

    • Responsive layouts out of the box 📱💻

    • Simplifies cross-browser compatibility & testing 🌐🧪

    • Great for team collaboration 🤝

    How to Use

    CSS Frameworks


    • Import the framework into your site:

      • <!doctype html>
      <html>
      <head>
      <meta charset="UTF-8">
      <meta name="viewport" content="width=device-width, initial-scale=1.0">
      <script src="https://cdn.tailwindcss.com"></script>
      </head>
      </html>


    • Use the styles provided by the framework:

      • <h1>Hello world!</h1>

      <h1 class="text-3xl font-bold underline text-pink-500 background-pink-50">
          Hello world!
      </h1>
