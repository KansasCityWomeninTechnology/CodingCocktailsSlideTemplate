Modules, Variables, & Operators


JavaScript



  • Official name: ECMAScript

  • Can change HTML content and CSS styling on a webpage

  • React to user interactions

  • Send network requests, etc

So what is ES6?



  • 2015 - new version of JavaScript was released called ES6 (or ES2015)

  • Major version that introduces many new features into JavaScript

  • Since ES6, a new version of JavaScript has been released every year

New Features in ES6



  • New variable declarations (const and let)

  • Arrow functions

  • New array methods

  • AND MORE!!

We'll be going through the more commonly-used features today!


Modules



Modules

  • In a repo, we have many JavaScript files

  • To avoid repeating work, we will want to reuse code from one file into another file

  • DRY Programming (Don't Repeat Yourself)

  • This "reuse" of code is done through exporting different files

Exporting before ES6



// One JavaScript file: ./sum.js

function sum(a, b) {
    return a + b;
};

module.exports = sum; // Export the function `sum`

// In another JavaScript file

import { sum } from "./sum.js"; // Imports the function `sum`

Now ES6



// One JavaScript file: ./sum.js

function sum(a, b) {
    return a + b;
};

export { sum }; // Exports the function `sum`

// In another JavaScript file

import { sum } from "./sum.js"; // Imports the function `sum`

We can also import/export multiple functions at once

// One JavaScript file: ./sum.js

function sum(a, b) {
    return a + b;
};

function multiply(a, b) {
    return a * b;
};

export { sum, multiply };
// Exports the functions `sum` and `multiply`
// In another JavaScript file

import { sum, multiply } from "./sum.js";
// Imports the functions `sum` and `function`

Variables



Variables before ES6



  • var declares a global-scoped variable

  • Allows access to a variable anywhere in the script after being declared

  • This was less secure as hackers could access variables

    • var drinks = 6;

Variables with ES6



  • Introducing let and const

  • let allows a variable's value to be reassigned

  • const declares a variable that cannot be reassigned

  • Both are local or block-scoped variables, which doesn't allow access to the value outside of scope

Variables tl;dr



  • Don't use var

  • Use const or let


Operators

Introduction



  • ES6 introduced Rest and Spread operators into JavaScript

  • Both use three dots…but for different purposes

  • Both help simplify array manipulation and function parameters

Spread Operator



  • Expands iterables such as arrays into individual elements

  • Allows us to merge, copy, or pass array elements to functions without explicitly iterating through them
const arr = [1, 2, 3];
// [1, 2, 3]

Spread Operators cont.



  • Goal: We want to create a new array that includes the existing array arr

Before spread operators:



const newArr = [5, 6, arr[0], arr[1], arr[2]];
// [5, 6, 1, 2, 3]

Now with spread operators:



const newArr = [5, 6, ...arr];
// [5, 6, 1, 2, 3]

To prove we need the spread operator:



const newArr = [5, 6, arr];
// [5, 6, [1, 2, 3]]

Spread Operators can be used to:



  • Combine or copy arrays

  • Combine objects

  • Pass arguments to functions

Combine Arrays:



const arr1 = [1, 2, 3];
const arr2 = [4, 5, 6];
const combinedArr = [...arr1, ...arr2];
// [1, 2, 3, 4, 5, 6]

Combine objects



const order = { name: "cookie", number: 12 };

const deliveryAddress = {
    street: "123 8th St",
    city: "Olathe",
    state: "KS",
}; const purchase = {...order, ...deliveryAddress};

// {name: "cookie", number: 12, street: "123 8th St", city: "Olathe", state: "KS"}

Without spread operators...

const order = { name: "cookie", number: 12 };

const deliveryAddress = {
    street: "123 8th St",
    city: "Olathe",
    state: "KS",
}; const purchase = {order, deliveryAddress};

// {
//   {name: "cookie", number: 12},
//   {street: "123 8th St", city: "Olathe", state: "KS"}
// }

Pass arguments to functions:



function sum(a, b, c) {
    return a + b + c;
}

const nums = [1, 2, 3];

const result = sum(...nums);
// 5

Rest Operator



  • Condenses elements into a single entity

  • By collecting remaining elements into a designated variable
const arr = [1, 2, 3, 4, 5, 6];

Rest Operators cont.



  • Goal: We want to destructure arr

Before rest operators:



const arr = [1, 2, 3, 4, 5, 6];

const first = arr[0]; // Extracting the first element
// first = [1]

const rest = arr.slice(1);
// Collecting the rest of the elements
// rest = [2, 3, 4, 5, 6]

After rest operators:



const arr = [1, 2, 3, 4, 5, 6];

const [first, ...rest] = arr;

// first = [1]
// rest = [2, 3, 4, 5, 6]

Data Types and Methods



Data Types we'll discuss



  • Strings

  • Arrays

  • Objects

Strings Before ES6:



  • Strings are written with quotes (single or double)

    • let text = "Hello";
    let text2 = 'Hello';
  • Use + to combine strings

    • let numCookies = "10";
    console.log("I plan to eat" + " " + numCookies + " cookies!");
    // I plan to eat 10 cookies!

ES6: String Literals



  • Strings literals allow us to access variables inside of a string without using the +

  • ` (backtick) represents the string

  • ${} represents the js variable

    • let numCookies = "10";
    console.log(`I plan to eat ${numCookies} cookies!`);
    // I plan to eat 10 cookies!

Objects



  • In JavaScript, almost "everything" is an object

  • Objects are called POJO (Plain Old JavaScript Object)

POJO



  • Objects are a key/value data store

  • Key (aka property) will always be stringified behind the scenes)

  • Value can be anything (string, number, another object, function, etc.)

Object Example



const mimosa = {
    alcohol: "champagne",
    juice: "orange",
    garnish: "orange peel",
};

Objects cont...



  • Call the value of an object's property:

console.log(mimosa.juice); // orange

Objects cont...



  • Can also use template literals:

console.log(`Put the ${mimosa.juice} in the ${mimosa.alcohol} and shake it all up!`);
// Put the orange in the champagne and shake it all up!

Object Destructuring



  • Beautiful thing in JS, so pay attention

  • Extracts values from an object property and assigns them to a variable

  • By default, the extracted object key name will become the variable

Object Destructuring Example



const mimosa = {
    alcohol: "champagne",
    juice: "orange",
    garnish: "orange peel",
};

Destructure the property:

const { alcohol, garnish } = mimosa;
console.log(`Add the ${alcohol} and top with the ${garnish}`);
const { alcohol, garnish } = mimosa;

Note we're not calling ${mimosa.alcohol} because destructuring "pulls" out the objects

Arrays and ES6



ES6 introduces some array helper methods:



  • indexOf

  • forEach

  • map

  • filter

  • find

indexOf



  • Allows us to get an index of an array item

  • If item is not found in an array, -1 is returned
const garnishes = ["orange peel", "lime wedge", "tahini"];

garnishes.indexOf("tahini");
// 3

garnishes.indexOf("mint");
// -1 because not in array

forEach



  • Iterates every single item in the array and performs a function

  • array.forEach((name) => {function(name)}
const garnishes = ["orange peel", "lime wedge", "tahini"];

garnishes.forEach((garnish) => console.log(`Garnish drink with ${garnish}));

// Garnish drink with orange peel.
// Garnish drink with lime wedge.
// Garnish drink with tahini.

map



  • Allows us to create a new array based on items in an existing array

  • array.forEach((name) => {function(name)}
const garnishes = ["orange peel", "lime wedge", "tahini"];

const drinkToppings = garnishes.map((garnish) => `${garnish} topping));

console.log(drinkToppings);
// ["orange peel topping", "lime wedge topping", "tahini topping"]

filter



  • Allows us to filter array items that meet certain conditions

  • Returns a new array that contains all items that meet the conditions

  • Can store the newly generated array in a new variable

Filter example

const garnishes = [
    {name: "orange peel", ready: true},
    {name: "lime wedge", ready: false},
    {name: "tahini", ready: true}
];

const completedGarnishes = garnishes.filter((garnish) => garnish.ready);

console.log(completedGarnishes);
// [
// {name: "orange peel", ready: true},
// {name: "tahini", ready: true}
// ]

find



  • Allows us to find array items that meet certain conditions

  • Returns the FIRST item in the array that meet the conditions

  • Similar to filter except returns only the first item and not all the items that meet the condition

Find example

const garnishes = [
    {name: "orange peel", ready: true},
    {name: "lime wedge", ready: false},
    {name: "tahini", ready: true}
];

const completedGarnishes = garnishes.filter((garnish) => garnish.ready);

console.log(completedGarnishes);
// {name: "orange peel", ready: true}

Functions



Function definition



  • A block of code designed to perform a particular task

  • The code is executed when it’s “called”

Function example

function makeCocktail(number) {
    console.log(`I am making ${number} cocktails.`);
}

// Call the function
makeCocktail(2);

// I am making 2 cocktails.

ES6 Arrow Functions



  • ES6 introduced arrow functions

  • "Regular" functions and Arrow functions are mostly the same

  • Difference is that regular functions have their own context, which arrow functions do not

  • Essentially, we're saving the variable as a function

Arrow Function example

function makeCocktail(number) {
    console.log(`I am making ${number} cocktails.`);
}

// Arrow function
const makeCocktail2 = (number) => {
    console.log(`I am making ${number} cocktails.`);
}

Arrow Function with no parameters

const makeCocktail2 = () => {
    console.log(`I am making cocktails.`);
}

Arrow Functions are Super Cool!


ES6 Default Parameters



  • Allows us to set a default parameter if no parameters are passed in a function

Default Parameters



  • Say we have the following method:

    • const serve = (drink) => {
        return `Here's your ${drink}!`;
    }

    serve("martini");
    // Here's your martini!

Default Parameters



  • Say we want to have a basic drink that can be automatically "served" when we call the method

    • const serve = (drink = "wine") => {
        return `Here's your ${drink}!`;
    }

    serve();
    // Here's your wine!
  • Notice that if we do NOT pass in a parameter, the "default" will be wine

Overwriting default parameters



  • Note that we still can override this default if we pass in a parameter

    • const serve = (drink = "wine") => {
        return `Here's your ${drink}!`;
    }

    serve("cosmopolitan");
    // Here's your cosmopolitan!

Anonymous Functions



  • A function without a name

    • function(a, b) {
        return a - b;
    }

Anonymous Functions



  • We then assign it to a variable to call the anonymous function later

    • let subtract = function(a, b) {
        return a - b;
    }
  • AKA, "Subtract" variable is calling the anonymous function

Arrow Functions vs Anonymous Functions?



  • Arrow functions are always Anonymous Functions

  • i.e. A variable is assigned to an anonymous function
    • const subtract = (a, b) => {
        return a - b;
    }
    const subtract = (a, b) => {
        return a - b;
    }

Arrow Functions vs Anonymous Functions?

Callbacks



  • Function that is passed as an argument to another function

Order of Execution


  • Functions will always execute in the order that they are called, not in the order that they are defined
const first = () => {
    console.log('Hello');
}

const second = () => {
    console.log('World');
}

second();
first();

// World
// Hello

Goal...



Let's say we want a calculation function that returns a sum of two numbers.


Then we want to log out that sum through a logging function.

Attempt #1


const displayLog = (value) => {
    console.log(value);
}

const calculator = (num1, num2) => {
    let sum = num1 + num2;
    return sum;
}

let result = calculator(5, 10);
displayLog(result);
// 15

Issues with this code



  • We must call two functions in order to show the result

  • So let's get rid of calling 2 separate functions...

Attempt #2


Call displayLog in our calculator function

const displayLog = (value) => {
    console.log(value);
}

const calculator = (num1, num2) => {
    let sum = num1 + num2;
    displayLog(sum);
}

calculator(5, 10);
// 15

Issues with this code



  • We cannot stop calculator function from logging the result

  • This is because we call displayLog inside of our calculator function

So what do we do????


Use Callbacks!



  • Remember: Callback is just a function that is passed in as a parameter

  • Allows you to execute calculator alongside a callback

  • i.e. calculator will run it's calculation function before running in the passed-in function

Callback Solution


// In this example, displayLog will act as a callback function const displayLog = (value) => {
    console.log(value);
}

// That is passed to calculator as an argument const calculator = (num1, num2, callbackFunction) => {
    let sum = num1 + num2;
    callbackFunction(sum);
}

calculator(5, 10, displayLog);
// 15

Wait what??



  • Remember: functions are objects in JavaScript

  • We are simply passing in displayLog function to our calculator function

  • This allows us to change how we want to handle the resulting sum

Leveraging Callback!


On the next page because it's a lot of code...

// One possible callback function
const displayLog = (value) => {
    console.log(value);
}

// Another possible callback function
const displayAlert = (value) => {
    console.log(value);
}

const calculator = (num1, num2, callbackFunction) => {
    let sum = num1 + num2;
    callbackFunction(sum);
}

// So we can now change how we display the sum (in a log or alert)
calculator(5, 10, displayLog); // logs out 15
calculator(5, 10, displayAlert); // alert popup showing 15

Synchronous vs Asynchronous



  • Synchronous will run operations in the order that operations are called

  • Asynchronous will run operations until an asynchronous function is reached, where executions will split off

  • An asynchronous function will be ran while the rest of the code executes

Synchronous Function Example

console.log(`a`);
console.log(`b`);

// a
// b

Async Functions



  • Where one function will be ran along with the rest of the code

  • Async will make a function return a promise (more on promises next section)

  • An asynchronous function will be ran while the rest of the code executes

  • Callback functions can be useful in async functions

  • Common async function: setTimeout()

setTimeout()


  • Specify a callback function to be executed after a certain number of milliseconds

  • setTimeout(someFunction, # milliseconds to wait to execute);

    • setTimeout(drink, 3000);

    function drink() {
        console.log("I'd like a cocktail!");
    }

    // Prints: "I'd like a cocktail!" after a 3000 millisecond (or 3 second) delay

Async Function example with setTimeout

// our async function
setTimeout(() => {
    console.log("Async");
}, 1000);

console.log(`a`);
console.log(`b`);

// a
// b
// Async

Notice that async log is called first, but is actually last to execute due to the delay
