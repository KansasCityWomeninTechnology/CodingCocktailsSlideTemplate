Introduction

What is a SPA?


Single Page Application


Single index.html page


Dynamically updates the page
based on user interaction

(without reloading the entire page!)


Better User Experience

'conventional' website



Different HTML pages (files)


Each URL serves up a different page


Files might share/import resources

...but that's as good as it gets


Single Page Application

You use them everyday

...but you probably don't know it!


Every URL updates a portion of the page


Obvious: Pages with scrollable sections


Less obvious: The URL changes but it's still served from a single index.html file


More app-like: Behaves more like an app on your PC or phone

SPA vs 'conventional' website

AJAX & Single Page Apps


Asynchronous JavaScript and JSON (or XML)


Data is exchanged between browser and server in the background


Requires JavaScript to be enabled in the browser (like SPAs in general)


SPAs require the brower to retain a single page
even when you need server communication

Synchronous vs Asynchronous

Python (sync) import requests

r = requests.get('https://api.github.com/events')
print(r.text)
print("I come after the request")


JavaScript (async): use xhttp or fetch var xhttp = new XMLHttpRequest();
xhttp.onreadystatechange = function() { ... };
xhttp.open("GET", "https://api.github.com/events", true);
xhttp.send();
console.log('I come after the request');

An async exercise

Single Page Application Cons


Anyone browsing without JavaScript enabled will not get all the functionality


Search Engine Optimization (SEO)

A data or JavaScript error can mean
you have no content for bots to crawl!


Workarounds:

- direct bots to prerendered content

- implement cache with prerender.io

Frameworks & Libraries

What's the difference?

  • Library: a set of functions that you can call
    • - Moment
    • - Lodash
    • - D3
    • - jQuery

  • Framework: imposes structure to your code
    & describes how your code should be written
    • - Angular
    • - Backbone

But then...

What's a platform?

A framework and then some: could include CLI tools, bundles of libraries, virtual environments, language packs, or more.

Libraries be like...

"Here's my number... so call me, maybe?"

Frameworks be like...

"Don't call me, I'll call you"

(number, string, boolean, etc)

Routing


Tells your application what to display for a URL


Main homepage route is '/'


Angular CLI adds the homepage route

...but we have to add the other routes


Other route examples:
'/about' or '/profile/:username'


View more on routing here

import { NgModule } from '@angular/core';
import { Routes, RouterModule } from '@angular/router';
import { UserProfileComponent } from './profile/profile.component';
import { AllMeetupsComponent } from './meetups/meetups.component';
import { LocalMeetupsComponent } from './local/local.component';
import { HomeComponent } from './home/home.component';
import { PageNotFoundComponent } from './404/404.component';

const routes: Routes = [
  { path: 'profile/:username', component: UserProfileComponent },
  { path: 'meetups', component: AllMeetupsComponent},
  {
    path: 'meetups-local',
    component: LocalMeetupsComponent,
    data: { zipcode: 64155 }
  },
  { path: '', component: HomeComponent },
  { path: '**', component: PageNotFoundComponent }
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule { }

Data Binding


Pass data around your application in either:

  • - one-direction (component -> view)
  • - bi-direction (logic <-> component)



The way data is bound depends on syntax

Data Binding


{{ }} data is sent to the view from the component

<p>My dog's name is {{user.dogsName}}</p>

This is typically where you'd use text in HTML


[ ] is a value taken from the component
and displayed in the view

<input [value]="user.firstName">

This is typically seen on attributes
in HTML template elements

Data Binding

( ) is a value taken from the view and passed to the component (user interaction w/ the page)

<button (click)="delete()">Delete</button>

This is typically seen on attributes
in template elements


[( )] is a bi-directional way to bind (share) data

<input [(ngModel)]="user.dogsName">

Takes data from the view and passes it to the component (& vice versa)

Adding data to a component


Start with a model: a structured way of representing an object


Import the model to the component


Setup or fetch initial data in OnInit, one of the component's lifecycle hooks


Finally, write HTML for the template
