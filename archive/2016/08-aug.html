Dependencies, Packages & Package Managers

What is a dependency?

  • Asset(s) that are required (used) by other assets
    • Frameworks/Libraries
    • Your own files
    • Single file
    • Bundle of files (images & code)
  • Real World Example: Cookie Dough
    • Flour, Eggs, Sugar
  • Code Example:
    • Without Dependencies
    • With Dependencies
  • All your assets (files) are dependencies that your application needs to run.
  • Frameworks/Libraries -- don't reinvent the wheel

Dependencies can have their own dependencies

  • Real World Example: Chocolate Chip Cookies
    • Chocolate Chips
    • Cookie Dough < Flour, Eggs, Sugar
  • Code Example: Slideshow w/bxSlider
    • my-slider.js < jquery.bxslider.js < jQuery.js

Why use dependencies?

Don't reinvent the wheel.
  • Modular, reusable code
  • Versioned

A package is...

a dependency that can be managed by a package manager.

May also be referred to as a module.

Types of Packages


  • Project Tools
    jQuery, Moment, Lodash

  • Development Tools
    Gulp, Sass, Yeoman

  • Tools for Life
    wallpaper-cli, itunes-remote (mac),
    is-pokemon-go-up-cli

A package manager is...

a glorious tool that allows you to manage your application's dependencies.

  • Installs packages
  • Upgrades packages
  • Removes packages
  • Manages a package's dependencies

Node.js

  • Created in 2009 by Ryan Dahl
  • Open-Source
  • Back-end JavaScript that is fast
    • JavaScript runtime
    • Built on Chrome's V8 JavaScript Engine
    • Written in C++
  • Cross-platform
    • Windows, Mac, Linux
  • More Node during a future session
  • open source - original source code is made freely available and may be redistributed and modified
  • Normally when we use `JavaScript`, we are referring to JS code that is written for the Front-end and executed by the browser (chrome). Node is server-side JS code.
  • The description from the Node site is ..
    • V8 JavaScript engine was written to make JavaScript in the browser more performant. (Also written in C++.) It takes JavaScript code & compiles it to native machine code (which varies based on machine/platform), so that it can run things faster (and that weren't possible before).

What is npm?

  • Created by Isaac Schlueter in 2009
  • Open-Source
  • npm is the default Package Manager for Node
    • Node is a dependency to use npm
      see what i did there?
  • Originally for managing dependencies in Node.js applications, it's expanded for management of:
    • Web Applications
    • Mobile Applications
    • Other JavaScript development tools & frameworks

Why npm?

  • Comes packaged with Node.js
  • Epic Registry
    • 280,000+ packages
  • > 4 million developers use npm every month
  • > 1 billion packages are downloaded every week
  • An npm package can contain any asset - HTML, JS, CSS
There are no security checks on packages so do some research before using a package the 1st time.

  • npm comes with Node, but may be out of date

Versioning - semver

  • 1.1.1
    • Patch | Bug fixes & minor changes
      • 1.0 or 1.0.x or ~1.0.4
    • Minor | New features, that don't break existing features
      • 1 or 1.x or ^1.0.4
    • Major | Changes that break existing features
      • * or x

* `~` tilde - Allows patch updates within current minor version
    ~2.2.3 up to, not including, 2.3.0

* `^` caret - Allows minor updates within current major version
    ^2.2.3 up to, not including, 3.0 (so 2.9)

Versioning - Set when installing

  • Most recent version (default)
  • Set via CLI 
    npm install <package>@<version>
    npm install moment@2.14.1
  • Set manually in package.json 
    
  "dependencies" : {
    "express" : "3.3.2"
  }

Using packages

In Front-End JS Files with Browserify

Step 1: Install

Install Globally (via CLI) [Personal Projects] 
npm install browserify -g

Install for Project Only (via CLI) [Team Projects] 
npm install browserify --save-dev
If you install globally, you can use for all your local projects (with 1 install). If you install per project w/ `--save-dev` flag, you have to do it for each project individually, but easier for other developers to get started on your project.
