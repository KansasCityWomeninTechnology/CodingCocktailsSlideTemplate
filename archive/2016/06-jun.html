HTML Highlights


<!--Every HTML requires the doctype declaration, then HTML tags -->
<!DOCTYPE html>
    <html>
     <!--The HEAD section contains meta data about the page -->
      <head>
        <meta name="name of your page"  author="yourname">
        <title>Browser tab title here </title>
      </head>
      <!--The Body section contains user content. -->
      <body>
        <h1>This is your heading.</h1>
        <p>Visible page content appears between body tags.</p>
      </body>
    </html>

Styling HTML with CSS


<!DOCTYPE html>
<html>
    <head>
    <!--The <link> in the HEAD section tells the browser
        where to look for the CSS stylesheet:-->
    <link type="text/css" rel="stylesheet" href="stylesheet.css"/>
    <title>Page Title</title>
    </head>
<body>
    <h1>heading content</h1>
    <p>Some Content here</p>
</body>
</html>

CSS Compilers

  • Browsers only understand CSS, but it has some limitations.
  • Compilers like Sass increase the flexibility of CSS with variables, functions, "mixins", and other features.
    • Variables: hold commonly used styling
    • Partials: Used for organization (i.e. separate documents for variables, typography, etc.)
    • Allows for nesting of your style
  • Sass files would then be compiled into regular CSS files that all web browsers could understand.
  • Manual Compilation (Command line) vs. Build Tool (i.e. Gulp)

jQuery Makes Adding Interacivity Easier

Adding jQuery to Your Page

Just as the <link> tag in the html page connects your CSS, a <script> tag connects your jQuery or JavaScript code


<script src="assets/scripts/jQuery.min.js"></script>

<script src="assets/scripts/script.js"></script>

NOTE: Order of script tags is important!

If your script file depends on another script file (as our script file depends jQuery), the dependancy has to be loaded prior to the script that depends on it so in our case the jQuery script tag has to come before our script tag.

How jQuery Makes Sites Interactive

Event - a user-initiated activity such as mouse click, hovering over an area, or a keystroke

  1. The user triggers an 'event' by clicking the mouse, hovering (mouseover) or through a keystroke.
  2. The script tag in the HTML document points to the *.js document where jQuery captures the instructions for what to do when the event happens.
  3. In the *.js document, an element from the HTML is selected using $('')
  4. When the 'event' is triggered related to the element jQuery can reacts by changing the CSS Styling, showing or hiding content, etc.
