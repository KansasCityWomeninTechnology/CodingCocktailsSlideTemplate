    Modules



    Organizes your app into
    related sections of functionality


    Root module is the very top
    small apps only need one


    Larger apps may have feature modules - each related to specific workflow/capabilities

    Components


    Every app has at least one root

    Application is a tree of components

    Each component contains:
    template, data, behavior

    Components

    <app-quiz></app-quiz> @Component({
      selector: 'app-quiz',
      templateUrl: './quiz.component.html',
      styleUrls: ['./quiz.component.css']
    })

    Lifecycle of a Component


    "Hooks" are events that signal a change in the component's lifecycle


    Main lifecycle hook is ngOnInit


    - Called once shortly after Angular creates the component

    - Commonly used for fetching the initial data needed for the component

    Read more here

    Routing


    Tells your application what to display for a URL


    Main homepage route is '/' and is created by Angular CLI


    ...but we have to add the other routes

    Other route examples:
    '/about' or '/profile/:username'


    View more on routing here
