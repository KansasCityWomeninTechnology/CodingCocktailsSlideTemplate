    SPA Introduction

    What is a SPA?


    Single Page Application


    Single index.html page


    Dynamically updates the page
    based on user interaction

    (without reloading the entire page!)


    Better User Experience

    'conventional' website



    Different HTML pages (files)


    Each URL serves up a different page


    Files might share/import resources

    ...but that's as good as it gets


    Single Page Application


    You probably use them everyday

    ...but you might not even know it!


    Every URL updates a portion of the page. Behaves more like an app on your PC or phone


    Single Page Application


    Obvious: Pages with scrollable sections


    Less obvious: The URL changes but it's still served from a single index.html file

    Modularity


    Make reusable pieces (components) that can be accessed across page



    • - Header/Navigation Bar
    • - Footer

    • - Slideshow

    Time to get Angular


    Choosing a framework or library

    Why did we choose Angular?


    CLI helps scaffold our project quickly


    Angular is very opinionated
    so this reduces architectural decisions


    Good documentation


    Angular uses TypeScript


    Made by Microsoft


    .ts file extensions instead of .js


    Transpiles into JavaScript


    Can write JavaScript in a TypeScript file
    (but not the other way around!)

    Why Use TypeScript?


    JavaScript is loosely typed
    a variable's value is flexible


    var drink = "Tom Collins";
    drink = 3;

    Why Use TypeScript?


    TypeScript provides static "types" (number, string, boolean, etc) for JavaScript

    var drink = "Mojito";
    drink = true; //ERROR!
    drink="Long Island";

    A variable's type is enforced, but not the value.

    Angular CLI

    (Command Line Interface)


    Check out the documentation here


    CLI does things the "right" way
    according to Angular style guide


    Install via npm npm install -g @angular/cli

    What can we do with the CLI?


    Create a new project ng new PROJECT-NAME-HERE --routing


    Serve your project
    (so we can see it in the browser) ng serve

    What can we do with the CLI?


    Scaffold components, services, directives, etc.


    Component ng generate component MY-NEW-COMPONENT ng g component MY-NEW-COMPONENT

    Service ng g service MY-NEW-SERVICE

    CLI Scaffolding

    What can we do with the CLI?


    Build: create dist directory w/ build artifacts ng build


    Install angular-cli-ghpages (one time only) npm install -g angular-cli-ghpages

    What can we do with the CLI?


    Build to your GitHub repo ng build --prod --base-href "https://yourusername.github.io/reponame/"


    And deploy! angular-cli-ghpages
