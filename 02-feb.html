What is CSS?



Cascading Style Sheets


Controls how elements look


Indicates style (font, color, size, position)


CSS3 is the latest version of CSS

Why do we care?

How do you use CSS?


Create a .css file


Ok, but now what?


Use any of the following 3 methods...

Method 1: Inline


<h1 style="color:orange;">My header</h1>

My header

Method 2: Internal

<!DOCTYPE html>
<html>
  <head>
    <style>
       h1 {color: orange;}
     </style>
   </head>

    <h1>My header</h1>
  </body>
</html>

My header

h1 {color: orange;} <!DOCTYPE html>

</html>

Special Selectors


  1. Children (direct) ul > li { background-color: yellow; }
  2. Descendant (any) ul li { background-color: yellow; }
  3. Adjacent (immediate sibling) div + p { background-color: yellow; }
  4. Attribute a[target="_blank"] { background-color: yellow; }

Pseudo Classes

/* unvisited link */
a:link {
  color: #ff0000;
}

/* visited link */
a:visited {
  color: #00ff00;
}

/* mouse over link */
a:hover {
  color: #ff00ff;
}

/* selected link */
a:active {
  color: #0000ff;
}

What is a CSS Compiler?



(a.k.a. pre- and post- processors)


Allows you to write faster & cleaner CSS


But Browsers only support CSS...

so you use a different syntax and compile it!

Pre-processed code -> CSS

Why use a pre-processor?


Variables giving styles a name
Nesting define styles within another style
Math ceiling / floor, rounding, min / max
@import link one CSS file to another
Mixins perform computations on inputs
Minification compress the CSS like a zip
Loops repeat color / position changes
@extend style A plus style B

Why We like Sass

Several pre-processors to choose from

  1. Less
  2. Stylus
  3. Sass


Each are slightly different (features & syntax)


Sass is the most common


Sass offers syntax flexibility based on file ext...
